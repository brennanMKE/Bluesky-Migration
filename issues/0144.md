## Description

The React Native post share menu includes **Copy link to post** (puts the bsky.app URL on the clipboard) and an implicit **Open in browser** (the web profile link). The SwiftUI `PostCard` exposes a `ShareLink` for native sharing and an ellipsis menu with Copy text / Mute / Report — but nothing for copying just the post URL or opening the post on the web.

## Steps to reproduce

1. Open the ellipsis menu on any post.
2. Look for Copy link / Open in browser.
3. Observe neither exists.

## Expected behavior

Add to the ellipsis menu:

- **Copy link to post** — copies `https://bsky.app/profile/<handle>/post/<rkey>` to the clipboard. Show a confirmation toast.
- **Open in browser** — opens the same URL via `openURL`.

The native ShareLink is fine to keep alongside; these two add the explicit copy + browser paths RN has.

## Actual behavior

ShareLink only; no copy-link or open-in-browser actions.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/com/post-controls/ShareMenu.tsx`.
- URL helper: `https://bsky.app/profile/\(handle)/post/\(rkey)` where `rkey` is the last component of `post.uri`.
- File to edit: `BlueskyKit/Sources/BlueskyUI/PostCard.swift` (ellipsis menu).

## Root cause

The SwiftUI `PostCard` ellipsis menu (added in #0143) only carried `Copy post text`, `Translate post`, `Mute thread`, and `Report post`. It exposed a system `ShareLink` for sharing but had no way to (a) put just the post URL on the clipboard, or (b) open the post in the user's default web browser. Both are first-class affordances in the RN client (`ShareMenuItems.tsx` → `Copy link to post`, plus the implicit "open on web" path).

## Fix

In `PostCard.swift`:

- Added `@Environment(\.openURL)` and a `showCopiedToast` `@State` flag.
- Added a single `copyStringToClipboard(_:)` helper that wraps the existing `UIPasteboard` / `NSPasteboard` branches, and re-routed the existing `Copy post text` button through it.
- Added two new ellipsis-menu items adjacent to `Copy post text` (RN order):
  - `Copy link to post` (`link` SF Symbol) — calls `shareURL(for:)` (already present), copies the resulting `https://bsky.app/profile/<handle>/post/<rkey>` to the pasteboard, and flips `showCopiedToast`.
  - `Open in browser` (`safari` SF Symbol) — opens the same URL via `openURL`.
- Both items are `.disabled` when `shareURL(for:)` returns nil (defensive — should not happen for real feed posts).
- Attached `.toast(isPresented: $showCopiedToast, message: "Copied to clipboard")` to the `PostCard` body so the confirmation matches RN's `Toast.show("Copied to clipboard")`.

The string `Copy link to post` matches RN's `_(msg\`Copy link to post\`)` exactly. `Open in browser` is a SwiftUI-side label (RN web has this implicitly via tab-open; native needs the explicit verb).

## Files changed

- `BlueskyKit/Sources/BlueskyUI/PostCard.swift`

## Gotchas

- `shareURL(for:)` already existed for the action-bar `ShareLink` (added in #0007). Reusing it keeps a single source of truth for the post URL format — if RN ever changes the bsky.app URL shape, only one helper updates.
- The toast modifier (`BasicComponents.swift` → `View.toast(...)`) is reused as-is; no new toast plumbing was needed.
