# 0003 — Feed list ignores Dark Mode — white background with black text

| | |
|---|---|
| **Status** | resolved |
| **Module** | BlueskyFeed |
| **Platform** | macOS |
| **First seen** | 2026-04-29 |

## Description

With macOS Dark Mode enabled the home feed renders with a white background and black text. The sidebar and navigation chrome adapt correctly to dark mode, but the post list area does not. The Bluesky website shows a dark background with light text in the same OS appearance.

## Steps to reproduce

1. Enable Dark Mode in macOS System Settings → Appearance.
2. Launch the app and sign in.
3. Observe the home feed post list.

## Expected behavior

The post list background is dark (matching the system window background color) and post text is light-colored, consistent with the rest of the macOS dark-mode UI and the Bluesky web reference.

## Actual behavior

The post list has a white background with black text regardless of the system appearance setting.

## Notes

The sidebar (`NavigationSplitView` leading column) and toolbar render correctly in dark mode, so the issue is isolated to the `List` / `ScrollView` used for the feed. Likely causes:

- An explicit `Color.white` or `NSColor.white` background applied to the list or row container, overriding the adaptive system color.
- A `listRowBackground` or `.background(.white)` modifier that doesn't adapt.
- The `List` style defaulting to `.plain` or `.inset` with a non-adaptive background on macOS — may need `.background(Color(.windowBackgroundColor))` or simply `.background(.background)` removed/replaced.

Start by inspecting `Sources/BlueskyFeed/FeedView.swift` and any row view in `Sources/BlueskyFeed/` for hardcoded color values.

## Resolution (2026-04-29)

Root cause: `BlueskyTheme`'s environment default is always `.light` (hardcoded in
`EnvironmentValues`), and no call site in the app injected a dark theme. `PostCard`
uses `theme.colors.background` (white in `.light`) as an explicit `.background()`
modifier, overriding any system-adaptive background the `List` would provide.

Fix applied in two files:

- **`Sources/BlueskyUI/Theme.swift`**: Added `adaptiveBlueskyTheme()` public view
  modifier backed by `AdaptiveBlueskyThemeModifier`. It reads
  `@Environment(\.colorScheme)` at render time and injects `BlueskyTheme.dark` when
  the OS is in dark mode and `BlueskyTheme.light` otherwise.

- **`Sources/BlueskyFeed/FeedView.swift`**:
  - Applied `.adaptiveBlueskyTheme()` to the top-level `VStack` in `body` so the
    correct theme cascades to `PostCard` and all child views.
  - Added `.listRowBackground(Color.clear)` to each `PostCard` row so the `List`'s
    own adaptive background shows through rather than being blocked by the row
    default white fill on macOS.

`swift build` passes cleanly after the change.
