# 0111 — Message thread doesn't render system messages (e.g. "@alice added @bob")

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyMessages |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

Both 1:1 and group threads can include system events that look like messages but aren't authored by anyone: "@alice left the group", "@bob added @carol", "Conversation locked", "Group name changed to <new>", message-deleted tombstones, etc. React Native renders these as italicized centered text (no bubble, no avatar). The SwiftUI thread loop only handles `MessageView` and skips other variants of the discriminated union — system events don't render at all, breaking the narrative of the conversation.

## Steps to reproduce

1. Open a group thread where members have been added/removed since you last opened it.
2. Compare to the same thread in RN.
3. Observe SwiftUI silently drops the system events while RN shows an italic line for each.

## Expected behavior

- The thread iterator handles all variants of `chat.bsky.convo.defs#message*` — `messageView`, `deletedMessageView`, and any group-events variants.
- For system events / tombstones: render as a centered italic small-text row in the secondary text color, no bubble, no avatar. Examples:
  - "Message deleted"
  - "@alice left the group"
  - "@bob added @carol"
  - "@op renamed the group to <new>"
- For unknown variants, render a generic "Unsupported message" placeholder rather than dropping silently.

## Actual behavior

Anything that isn't a `MessageView` is dropped. Group threads look incomplete.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Messages/components/SystemMessageItem.tsx`.
- Lexicon variants: see `chat.bsky.convo.defs` for `deletedMessageView`. Group events may surface as their own discriminator under `chat.bsky.group.*` (added 2026-04-26 — see Progress.md drift note).
- File to edit: `BlueskyKit/Sources/BlueskyMessages/MessageThreadScreen.swift` (the messages loop ~lines 74–77) — switch on the discriminated message union, dispatch to `MessageBubble` or `SystemMessageRow`.
