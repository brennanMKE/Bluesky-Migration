# 0013 — Module 3 gate: preferences persistence and cache stale-while-revalidate not validated

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyDataStore |
| **Platform** | All |
| **First seen** | 2026-04-29 |

## Description

The Module 3 gate — preferences survive app restart and `SwiftDataCacheStore` serves stale content while a fresh fetch loads — has not been validated in a running app.

## Steps to reproduce

1. Build and run the app.
2. Change a preference (e.g. theme) in Settings.
3. Force-quit and relaunch.
4. Confirm the preference is applied on launch.
5. Navigate to the home feed and confirm cached posts appear immediately before the network response arrives.

## Expected behavior

Preferences are persisted via `UserDefaultsPreferencesStore` and restored on next launch. Cache entries are served immediately and refreshed in the background.

## Actual behavior

Not validated. Implementation exists but no gate run has been performed.
