# 0136 — Content filter settings show only global labels; need per-labeler preferences

|---|---|

## Description

`ContentFilterSettingsScreen` exposes a fixed set of global Bluesky labels (porn, sexual, nudity, graphic-media, hate, spam) with hide/warn/show pickers. The React Native reference builds the screen dynamically from each subscribed labeler's `labelValueDefinitions`:

- Each labeler gets its own section.
- Each section lists every label the labeler defines (which can be custom — e.g., a labeler might define `politics`, `news-outlet`, etc.).
- Per-label visibility picker writes to `ContentLabelPref` keyed by `(labeler DID, label name)`.
- Some labels are "self-applied only" or "system" — those rows are read-only.

The SwiftUI version cannot configure custom labeler labels; only the built-in global set works.

## Steps to reproduce

1. Subscribe to a custom labeler (one that defines its own labels).
2. Open Content Filters.
3. Observe the labeler's custom labels are absent.

## Expected behavior

`ContentFilterSettingsScreen` becomes per-labeler:

- One section per subscribed labeler (loaded from `subscribedLabelers` per #0133).
- Each row in a section is a label defined by that labeler, with hide/warn/show picker.
- Built-in Bluesky labels render under a "Bluesky Moderation" labeler section.
- Per-row visibility writes to `ContentLabelPref(label, visibility, labelerDid)`.

## Actual behavior

Six global labels, hardcoded.
