## Description

Manual sign-off for the app shell — tab bar (iOS / iPad), sidebar
(macOS), top-right toolbar, deep-link routing, push-notification
routing, and back-stack behaviour across tab switches.

## Test sequence — Tab bar / sidebar

1. **iPhone: bottom tab bar.** Launch on iPhone. Confirm five tabs
   in this order: Home · Search · Messages · Notifications ·
   Profile. Confirm even spacing across the bar.
2. **iPhone: tab badges.** With unread messages and notifications,
   confirm badges appear on the respective tab icons.
3. **macOS: sidebar.** Launch on macOS. Confirm a left sidebar
   with the same five entries (plus a Saved entry — #0060) and a
   refresh button in the toolbar. Selecting a sidebar row switches
   the detail.
4. **iPad regular: sidebar.** Same as macOS.

## Test sequence — Tab switching

5. **Stack reset on tab switch.** Navigate deep inside Home (open
   a thread → open a profile → open another thread). Switch to
   Search. Switch back to Home. Confirm the stack reset to the
   home feed root (#0055).
6. **macOS: switch from thread to another tab.** While viewing a
   thread on Home, click Messages in the sidebar. Confirm the
   active tab visibly changes to Messages, not stuck in the
   thread view (#0055).
7. **Per-tab state isolation.** Open a deep view in Notifications,
   switch tabs, return to Notifications. Confirm the deep view
   is gone (root list) — not preserved across tab switches.

## Test sequence — Top-right toolbar (macOS)

8. **Toolbar items.** Confirm the top-right toolbar shows three
   icons cleanly spaced (not crammed into a single capsule —
   #0148): compose pencil, saved feeds, and refresh.

## Test sequence — Deep links

9. **`bluesky://` URL scheme.** With the app closed, run from
   another app: open URL `bluesky://profile/bsky.app`. Confirm
   the app launches and navigates directly to that profile.
10. **In-app deep link.** With the app open, repeat #9. Confirm
    it routes within the existing window, not opening a new
    window (#0052).
11. **Post deep link.** Open `bluesky://profile/bsky.app/post/<rkey>`.
    Confirm it pushes the thread for that post.
12. **Web URL deep link.** Open `https://bsky.app/profile/bsky.app`
    from another app. Confirm same behaviour as the custom scheme.
13. **Bad URL handling.** Open a malformed URL like
    `bluesky://nothing-here`. Confirm the app does not crash and
    surfaces an error or no-op gracefully.

## Test sequence — Push routing (iOS only)

14. **Push tap → thread.** With push notifications enabled, receive
    a like / reply / mention notification. Tap the banner.
    Confirm the app opens and navigates to the relevant thread
    (#0030 — currently blocked on Apple Developer setup).
15. **Push tap → profile.** Same with a follow notification —
    confirm navigation to the follower's profile.

## Test sequence — Sidebar drawer (iOS, if present)

16. **Open sidebar drawer.** On iPhone, swipe from the left edge
    (or tap the avatar in the top-left). Confirm the side drawer
    appears with: profile header (avatar / name / handle), full
    nav (Home / Search / Messages / Notifications / Profile /
    Saved / Settings / Help), footer legal links (#0150).

## Sign-off checklist

- [ ] iPhone tab bar order, spacing, badges
