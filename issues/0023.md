# 0023 — Module 8 feature: verified badges and labeler badges not shown on profiles

| | |
|---|---|
| **Status** | resolved |
| **Module** | BlueskyProfile |
| **Platform** | All |
| **First seen** | 2026-04-29 |

## Description

`ProfileHeaderView` does not display verification badges or labeler badges next to the display name or handle. These are present in the React Native app for accounts that have labels applied by labelers or are verified handles.

## Steps to reproduce

1. Navigate to a profile with a known verification badge (e.g. an official Bluesky account or a labeler).
2. Observe the display name area.

## Expected behavior

A badge icon (checkmark or custom labeler badge) appears next to the display name for verified or labelled accounts.

## Actual behavior

No badges are shown regardless of the account's label status.

## Notes

Labels come from `ProfileDetailed.labels: [Label]`. The badge rendering logic should check for `verification` or labeler-specific label values and render an appropriate `Image(systemName:)` or custom asset.
