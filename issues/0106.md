# 0106 — Message bubbles have no per-message context menu

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyMessages |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

In React Native, long-pressing (or right-clicking on web) any message bubble opens a context menu with: **React** (emoji picker), **Copy**, **Translate**, **Delete** (own messages only), and **Report** (others' messages). The SwiftUI `MessageBubble` has no gesture handlers and no menu — taps and long-presses do nothing. There's no way to copy a message, delete a sent mistake, or report abuse from the thread surface.

## Steps to reproduce

1. Open any message thread in the SwiftUI app.
2. Long-press a message bubble.
3. Observe nothing happens.

## Expected behavior

Long-press (iOS) or right-click (macOS) on any bubble opens a context menu:

- **React** — opens reactions picker (covered by #0110 if filed).
- **Copy** — copies message text to pasteboard.
- **Translate** — opens system Translate sheet (iOS 17+) or a translate-to-clipboard fallback.
- **Delete** — only on own messages; calls `chat.bsky.convo.deleteMessageForSelf` (or appropriate endpoint) and removes locally.
- **Report** — only on others' messages; opens existing `ReportDialog` from `BlueskyModeration` with the message as subject.

iOS uses `.contextMenu { … }`. macOS uses the same modifier — SwiftUI bridges to AppKit's right-click automatically.

## Actual behavior

No menu at all on long-press / right-click.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Messages/components/MessageItem.tsx` (search for `MessageContextMenu`).
- Endpoints needed: `chat.bsky.convo.deleteMessageForSelf` (already in `BlueskyCore`?), translation can use `TranslationKit` on iOS 17+ / system translation services.
- File to edit: `BlueskyKit/Sources/BlueskyMessages/MessageThreadScreen.swift` (the `MessageBubble` view).
- Pairs naturally with reactions (#0110) since both attach to the bubble.
