# 0130 — Video feed has no interactive overlays (mute, like, comment, share, profile)

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyFeed |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

`VideoFeedView` renders a vertical paging `TabView` of `AVPlayer`-backed videos with an avatar, handle, bio, and post text overlay. It is missing the interactive controls the React Native video feed has (TikTok-style):

- **Mute toggle** (top-right or bottom-right) — videos default muted; users tap to unmute.
- **Like / repost / reply / share** action buttons stacked vertically on the right edge, with counts.
- **Comments / replies** entry — opens the post's thread.
- **Avatar/handle** is tappable to navigate to the author's profile.
- **Scrubber** at the bottom showing playback progress.
- **Tap to pause** in the center — RN supports a long-press / tap-to-pause gesture overlay.

Without these, the video feed is a passive viewer with no engagement affordances.

## Steps to reproduce

1. Open the video feed.
2. Look for like / comment / share buttons or a mute toggle.
3. Observe none.

## Expected behavior

Add to `VideoPostPage`:

- A muted/unmuted icon button (top-right corner) tied to the player's `isMuted`.
- A right-aligned vertical stack of action buttons (like, repost, comment count, share) with counts; reuse the post mutation paths from `FeedViewModel`.
- Avatar + handle row tappable into `ProfileScreen`.
- Bottom scrubber bar showing video progress (read `player.currentTime` via `addPeriodicTimeObserver`).
- Single-tap on the player surface toggles play/pause.

## Actual behavior

Player + static avatar/text only; no interactive overlays.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/VideoFeed/components/`.
- File to edit: `BlueskyKit/Sources/BlueskyFeed/VideoFeedView.swift`.
- Reuse `BlueskyUI` post-action button components for like/repost/share so the video feed and the regular feed share state.
