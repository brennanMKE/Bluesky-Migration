## Description

The iOS top bar (#0072) introduces a `#` button that, in the React Native reference, navigates to a "My Feeds" screen. That screen does not yet exist in the SwiftUI app and needs to be built. It is the home for managing custom feeds — the user's pinned feeds, saved feeds, suggested feeds, and a search/discovery entry point. It is one of the most-used screens in the RN client because it's the only way to switch the active feed beyond the small set in the Home tab strip.

## Reference

`Bluesky-ReactNative/src/view/screens/Feeds.tsx` (or `MyFeeds.tsx` depending on version) is the canonical implementation. Cross-reference `src/state/queries/preferences/savedFeeds.ts` for the data model.

## Scope

### Sections

The screen is a single scrollable view with these sections, in order:

1. **Search bar** — type-ahead search against the feed-discovery API (`app.bsky.unspecced.getSuggestedFeeds` plus `app.bsky.feed.searchPosts`-style fallback). Tapping a result opens that feed.
2. **My Feeds** — the user's saved feeds (pinned + saved-but-unpinned). Pinned feeds appear first with a pin indicator; user can drag to reorder, swipe to unpin, or tap to open. Reorder persists via `app.bsky.actor.putPreferences` saved-feed entries.
3. **Discover new feeds** — a curated list of suggested feeds from `app.bsky.unspecced.getSuggestedFeeds`. Each card shows feed name, description, author, like count, and a "+ Add" button.

### Interactions

- **Tap a feed card → open the feed timeline** in a new navigation destination using the existing `FeedView` (or the equivalent timeline view).
- **Pin / unpin toggle** on each saved feed.
- **Drag-to-reorder pinned feeds** with haptic feedback on iOS.
- **Swipe-to-delete** unpinned saved feeds.
- **"+ Add"** on a discover card adds the feed to saved feeds (unpinned by default).

### State

- `SavedFeedsViewModel` already exists in `BlueskyFeed/SavedFeedsScreen.swift` plus stores. Reuse rather than duplicate.
- Suggested-feeds list lives in a new `SuggestedFeedsStore` (or as a method on the existing `SavedFeedsStore`).
- Search results list is its own ephemeral state in the view model.

## Implementation notes

- Place this screen in `BlueskyKit/Sources/BlueskyFeed/MyFeedsScreen.swift` since it's primarily a feed-management surface.
- Wire it as the `navigationDestination` for the top-bar `#` button added in #0072.
- iOS-only consideration: use `.draggable` / `.dropDestination` for reordering and `.swipeActions` for unpin. macOS may need a different reorder affordance — confirm before shipping.
- Use the existing component patterns from `SavedFeedsScreen` and `BookmarksScreen` for visual consistency.

## Acceptance

- New `MyFeedsScreen` view exists and is reachable via the top-bar `#` button.
- Search bar returns live suggestions as the user types.
- My Feeds section shows pinned + saved feeds; pin-state and ordering survive force-quit and relaunch via server preferences.
- Discover section loads suggested feeds and the "+ Add" flow works end-to-end.
- iOS Simulator and macOS builds pass.

## Open questions

- **Search backend.** Confirm whether feed search has a dedicated lexicon or piggybacks on the suggested-feeds endpoint. The RN code is the source of truth.
- **Reorder ergonomics on macOS.** SwiftUI drag-to-reorder on macOS sometimes lags; may need a `↑↓` button affordance instead.
