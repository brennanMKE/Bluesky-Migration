# 0142 — Posts have no "Show more" / "Show less" expansion for long text

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyFeed / BlueskyUI |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native client clamps long post bodies to ~25 lines (`MAX_POST_LINES`) and shows a **"Show more"** button below; tapping expands the full text and replaces the button with **"Show less"**. The SwiftUI `PostCard` / `PostBodyView` either renders the full body uncapped (in feed) or applies a fixed `lineLimit` set by the caller (in compact contexts) — but there's no expansion control either way. Long-form posts break the feed visually with massive cards.

## Steps to reproduce

1. Find a post with a body longer than ~25 lines.
2. Observe in feed it renders the entire wall of text, with no "Show more" affordance.

## Expected behavior

- `PostBodyView` clamps to a configurable max (default 25 lines for feed, unlimited for thread highlighted post).
- When clamped, a "Show more" button appears below the text.
- Tapping expands; the button becomes "Show less".
- Auto-expanded when post is the focal post in a thread.

Use `Text.lineLimit(_:)` plus a measure pass (e.g., `GeometryReader` + `PreferenceKey`) or SwiftUI's `truncationMode` + line-count helper.

## Actual behavior

No expansion control. Posts render at their full length in feed.
