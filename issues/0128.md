# 0128 — Starter Pack create is a single sheet; RN uses a multi-step wizard

## Description

`StarterPackCreateSheet` collects name, description, and a list selection in a single sheet. The React Native client uses a multi-step wizard at `Bluesky-ReactNative/src/screens/StarterPack/Wizard/`:

1. **Profiles** — actor picker; the user searches and selects accounts to include directly (without needing to first build a list).
2. **Feeds** — feed picker; up to N feeds to include.
3. **Details** — name, description, optional banner.
4. **Preview** — preview how the starter pack will appear; final Create button.

The single-sheet version misses the feeds slot entirely and forces the user to pre-build a list before creating the pack.

## Steps to reproduce

1. Open the starter-pack create flow.
2. Compare to RN's wizard.

## Expected behavior

Refactor `StarterPackCreateSheet` into a `NavigationStack`-driven wizard with the four steps above. Each step has Next / Back. Profiles step uses an actor-search field with debounced lookup; Feeds step uses a feed-search field. Final preview step calls `com.atproto.repo.applyWrites` to create both the list (containing selected actors) and the starter pack record in one transaction.

## Actual behavior

Single sheet; feeds slot missing; requires a pre-existing list.
