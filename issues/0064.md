# 0064 — Module 12 gate (iOS): composer live validation on iPhone

|---|---|

## Description

Once the iOS build is restored (#0061), the Module 12 composer gate must be re-validated on a real iPhone. iOS uses a different image picker (`PHPickerViewController` via `PhotosPicker` / `PhotosPickerItem`) and different keyboard / safe-area / rubber-band scroll behavior than macOS, and several composer code paths are gated `#if os(iOS)` — they have not been exercised since #0034 landed.

This issue split off from #0035, which now focuses on the macOS gate.

## Steps to reproduce

1. Resolve #0061 so the iOS build is green.
2. Install the app on a registered iPhone (or run in iOS Simulator) and sign in.
3. Run the live checks below.

## Live checks (must all pass)

- **Plain-text post.** Type a short post, submit, confirm it appears in your feed.
- **Image post (PHPicker).** Tap the image button, pick 1–4 images via PHPicker, optionally add alt text, submit, confirm the post renders with all images in the feed and embeds round-trip after a refresh.
- **Image post — alt text.** With one image attached, open the alt-text editor, save, submit, confirm `accessibilityLabel` is preserved on the post embed.
- **Video post.** Tap the video picker, select a video, confirm it uploads, submit, confirm the post renders with the video.
- **@mention autocomplete.** Type `@` plus a few characters; confirm suggestion overlay appears; pick a suggestion; confirm the posted mention is a live profile link.
- **Link card preview.** Paste a URL; confirm the OpenGraph card resolves with title/description/thumbnail; submit; confirm the card embed appears in the feed.
- **Thread composer.** Add a second post to make a thread; submit; confirm both posts appear with the second as a reply to the first.
- **Draft persistence (per-context).** Type without posting; dismiss; reopen the same composer (root vs. reply-to); confirm the draft restored matches the context. Drafts are keyed per-`replyTo.uri`.
- **Draft clears after successful post.** Post successfully; immediately reopen the composer; confirm the text field is empty.
- **Keyboard / safe area.** Confirm the input bar tracks the keyboard, the character counter stays visible, and the toolbar is reachable above the home indicator.

## Expected behavior

All ten checks pass. iOS-specific paths (`PhotosPicker` / `PhotosPickerItem`, keyboard avoidance, safe-area insets) behave correctly without regressions from the macOS-focused work.

## Actual behavior

Cannot be validated until #0061 is resolved.
