# 0148 — macOS top-right toolbar shows three icons (compose / list-star / chevrons) crammed in a single capsule

## Description

The top-right of the macOS SwiftUI app shows three icons packed into a single rounded capsule: a compose square (pencil), a list-with-stars glyph, and a `>>` double-chevron. The grouping reads as "one button" until you look closely; the icons are unequally sized, the capsule has a thin border that makes them look like a segmented control, and there's no spacing between them.

There is no Mac RN reference. The iOS RN reference (`Bluesky-ReactNative`, see attachment) places the equivalent affordances individually:

- **Hamburger** at the very leading edge of the top bar (opens the sidebar drawer).
- **Butterfly logo** centered.
- **Hash (`#`)** at the trailing edge (opens "My Feeds" — see #0073).

The SwiftUI macOS top bar is conflating multiple concepts into one capsule, and the `>>` glyph in particular has no obvious purpose for the user — it's likely the macOS "expand sidebar" / "show inspector" affordance that has visually merged with the action cluster.

## Attachments

![SwiftUI macOS toolbar — three icons crammed in a capsule](swiftui-toolbar.png)

## Steps to reproduce

1. Run the SwiftUI app on macOS.
2. Look at the top-right toolbar of the Home (or any tab) screen.
3. Compare to the RN-iOS top bar in #0072 / #0073.

## Expected behavior

Either:

- **Match the RN-iOS top bar layout** even on macOS: hamburger leading, wordmark centered, `#` trailing — three independent buttons with normal toolbar spacing, no capsule wrapper. The `>>` (sidebar expand) button should be a system `ToolbarItem(placement: .navigation)` if needed and should be visually separated from the action cluster.
- Or, if a custom action cluster is intentional on macOS, give each icon: a clear label / tooltip, equal sizing, separator gaps, and remove the capsule outline. And replace `>>` with the standard SF Symbol for the sidebar control (`sidebar.left` or `sidebar.right`).

The middle icon (list-with-stars) is presumably "My Feeds" / saved feeds — that's the `#` button on iOS. Confirm and align the icon to the RN equivalent or use the same `#` glyph.

## Actual behavior

Three icons in a single capsule with no labels, unclear purpose, and a `>>` affordance that doesn't match any RN entry-point.

## Notes

- iOS top-bar pattern is `BlueskyTopBar` (added in #0072). macOS uses a different toolbar path — find it in `Bluesky-SwiftUI/Bluesky-SwiftUI/MainTabView.swift` (the `macOSSidebarLayout` or similar).
- Tooltips on every macOS toolbar button are tracked separately by #0009 (already resolved); confirm that issue's helper covers these too.
- Pairs with #0150 (sidebar drawer parity) — once the sidebar is the proper drawer affordance, the `>>` here may not be needed at all.
