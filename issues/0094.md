# 0094 — Deactivated-account screen is missing

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyAuth / Bluesky-SwiftUI |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

When a user signs in with an account whose status is `deactivated`, the React Native client shows a dedicated screen at `Bluesky-ReactNative/src/screens/Deactivated.tsx` with: an explanatory message, a "Reactivate your account" button (calls `com.atproto.server.activateAccount`), a "Sign out" button, and an account picker so the user can switch to a non-deactivated account. The SwiftUI client has no equivalent — at best the user lands on a broken/empty MainTabView, since `SessionManager` doesn't branch on the deactivated status.

## Steps to reproduce

1. Have a Bluesky account in deactivated state.
2. Sign in to the SwiftUI app with that account.
3. Observe what the app does (likely: blank/error main tab view, no clear path to reactivate).

## Expected behavior

When `SessionManager` detects the active session is on a deactivated account, the app routes (above MainTabView) to a `DeactivatedView` that:

- Displays a status banner ("Your account is deactivated").
- Shows a "Reactivate account" primary button → calls `com.atproto.server.activateAccount`, then re-bootstraps to MainTabView on success.
- Shows a "Sign out" secondary action.
- Optionally exposes the existing `AccountPickerView` so the user can switch to another account.

## Actual behavior

No deactivated handling. The session is treated as a normal logged-in session and the user lands on MainTabView with API calls failing (deactivated accounts can't read/write).

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Deactivated.tsx`.
- Detection: account status comes from `com.atproto.server.getSession`'s `status` field (`active` / `takendown` / `suspended` / `deactivated`). May need to be added to `Account` / `StoredAccount` in `BlueskyCore`.
- AT Proto endpoint: `com.atproto.server.activateAccount`.
- File to create: `BlueskyKit/Sources/BlueskyAuth/DeactivatedView.swift`; gate from `RootView` in the app target.
- Pairs with #0095 (takendown screen) — both branch off the same status field.
