# 0045 — Thread view styling is broken: collapsed rows, connector lines, dark mode failure

| | |
|---|---|
| **Status** | resolved |
| **Module** | BlueskyFeed |
| **Platform** | All |
| **First seen** | 2026-04-29 |

## Description

The thread (post detail) view has several major visual problems:

1. Replies are collapsed by default into tiny compact rows with a small avatar and 2-line text preview. The user must tap each reply to expand it — this is not how Bluesky works and makes the thread unusable.
2. A vertical connector line (left-side border) is rendered alongside replies, adding visual noise and indenting the content incorrectly.
3. The focal post has a white background even in dark mode, breaking the color scheme.
4. Overall the thread looks nothing like the main feed or the Bluesky reference app.

## Expected behavior

The thread view should match the Bluesky reference app:
- The focal post appears at the top as a full `PostCard`.
- All replies appear below in a flat list, each rendered as a full `PostCard` — same style as the home feed.
- No collapsed rows, no connector lines for the reply list, no expand-on-tap behavior.
- Dark mode is respected throughout.

## Actual behavior

Replies are collapsed into compact rows. There is a vertical connector line indenting reply content. The focal post has a white/light background in dark mode.

## Notes

Root cause: issue #0020 (inline post expansion) introduced a `collapsedPostRow` helper and the expand-on-tap pattern. The connector lines were added at the same time. Both should be removed. The fix is to rewrite `ThreadView` to use a flat `List` of `PostCard` items — focal post first, then all direct replies — matching the layout of `FeedView.list`.
