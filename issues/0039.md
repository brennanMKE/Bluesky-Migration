# 0039 — Tapping a post navigates to a blank screen

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyFeed |
| **Platform** | All |
| **First seen** | 2026-04-29 |

## Description

Tapping a post in the feed navigates to the thread screen, but the screen is blank — no root post, no replies, no loading indicator.

## Steps to reproduce

1. Launch the app and sign in.
2. Wait for the home feed to load.
3. Tap any post.

## Expected behavior

The thread screen opens and displays the tapped post as the root, with its reply tree below it.

## Actual behavior

The thread screen opens but shows a completely blank view with no content.

## Notes

Likely root causes to investigate in order:

1. **`ThreadViewModel` not triggered** — The `NavigationLink` or `navigationDestination` handler may not be calling `loadThread()` on the view model. Check whether `.task {}` or `.onAppear {}` on `ThreadView` is firing.

2. **`ThreadViewModel` is using the wrong post URI** — The URI passed via the navigation destination may be nil, empty, or malformed, causing `getPostThread` to return an error that is silently swallowed.

3. **Silent fetch error** — `ThreadViewModel.loadThread()` may be throwing (e.g. a decoding error similar to issue 0002) and resetting to an empty state without surfacing an error view.

4. **Missing `@State`/`@Environment` injection** — `ThreadView` may depend on a `network` or `accountStore` environment object that is not injected through the `navigationDestination` call site, causing a crash or no-op on load.

Add an error state view to `ThreadView` so failures are visible rather than silent.
