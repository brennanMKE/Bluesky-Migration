# 0101 — Composer doesn't warn when alt-text-required is on but images have no alt text

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyComposer |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native composer respects an "alt text required" preference (set in Accessibility settings). When that preference is on and the user has attached images without alt text, RN displays an `Admonition` error banner above the action bar reading approximately "All images must have alt text" and disables the Post button until alt text is provided.

The SwiftUI composer has the preference stored in `SettingsViewModel` (Accessibility) but doesn't consult it. A user with the preference enabled can still post without alt text, defeating the accessibility setting.

## Steps to reproduce

1. In Settings → Accessibility, enable "Require alt text".
2. Open the composer and attach an image.
3. Without typing alt text, tap Post.
4. Observe the post submits.

## Expected behavior

When `requireAltText == true` AND any attached image lacks alt text:

- A red admonition banner appears above the toolbar: "All images need alt text" (or RN's exact copy).
- The Post button is disabled until every image has non-empty alt text.
- Once all images have alt text, the banner clears and Post re-enables.

When the preference is off (default), behavior is unchanged.

## Actual behavior

Preference is ignored by the composer. Posts submit regardless of missing alt text.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/com/composer/Composer.tsx` — search for `requireAltTextEnabled` / `Admonition`.
- Preference key: `requireAltText` in `UserDefaultsPreferencesStore`. Already wired in `AccessibilitySettingsScreen` but not consumed by the composer.
- Files to edit: `BlueskyKit/Sources/BlueskyComposer/ComposerSheet.swift` (read preference + render admonition), `ComposerViewModel.swift` (computed property: `canPost` returns false when missing alt text).
