# 0043 — Pull-to-refresh not implemented for feeds, replies, and search

| | |
|---|---|
| **Status** | implemented |
| **Module** | BlueskyFeed / BlueskyProfile / BlueskySearch |
| **Platform** | All |
| **First seen** | 2026-04-29 |

## Description

Pull-to-refresh (swipe down from the top of a scroll view) has no effect or is not available in the home feed, the replies/notifications view, and the search results view. Users expect this standard gesture to reload content.

## Steps to reproduce

1. Open the Home feed, the Notifications tab, or the Search tab.
2. Swipe down from the top of the list to trigger a refresh.

## Expected behavior

Pulling down reveals a standard activity indicator and triggers a network fetch that reloads the list with fresh content.

## Actual behavior

The gesture has no effect, or the view lacks a `.refreshable` modifier entirely.

## Notes

The home `FeedView` already has `.refreshable { await vm.refresh() }` on its `List`, so the feed may be partially covered. Confirm it works end-to-end and check that the same pattern is applied to the replies/notifications list and the search results list. If those views use `ScrollView` instead of `List`, a `RefreshableScrollView` wrapper or `.refreshable` on a `List` container will be needed.
