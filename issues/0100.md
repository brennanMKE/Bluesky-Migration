# 0100 — Composer is missing a Drafts list / save UI

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyComposer |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native composer has a `DraftsButton` in the top toolbar that opens a list of saved drafts: each draft shows a preview, timestamp, and tap-to-resume / swipe-to-delete affordances. A separate "Save Draft" action lets the user save the in-progress composer state without posting. The SwiftUI composer has only auto-save-on-dismiss persistence (per #0034), which means the user has no way to see, manage, or load past drafts.

This is filed separately from #0034 because that issue covers draft *persistence* mechanics; this one covers the user-facing **drafts UI** (button + list + manage).

## Steps to reproduce

1. Open the SwiftUI composer, type a partial post.
2. Look for a way to save the draft explicitly, or to view past drafts.
3. Observe neither affordance exists.

## Expected behavior

- **Drafts button** in the composer top toolbar (RN places it to the left of the Post button).
- Tapping it opens a `DraftsSheet`:
  - Empty state if no drafts.
  - List of drafts: avatar + first 2 lines of text + relative timestamp + image-count badge if any.
  - Tap a draft to load it into the composer.
  - Swipe to delete a draft.
- **Save Draft** action: when the composer is dismissed via Cancel and the user has typed text, prompt "Save Draft / Discard / Cancel". The Save path stores the state in the existing persistence mechanism.
- **Auto-save** continues to write a single "in-progress" draft so a force-quit doesn't lose work.

## Actual behavior

No drafts button, no list, no explicit save action. The composer either submits or discards.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/com/composer/drafts/`.
- Storage: drafts can live in `UserDefaultsPreferencesStore` (small) or `SwiftDataCacheStore` (better — supports per-draft id + media references).
- Files to edit: `BlueskyKit/Sources/BlueskyComposer/ComposerSheet.swift` (toolbar + dismiss-with-text prompt), new `DraftsSheet.swift`, plus a `DraftsStore` in `BlueskyComposer` that wraps the cache.
