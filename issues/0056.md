# 0056 — Profile screen missing banner, avatar placeholder broken, post cards not dark mode

| | |
|---|---|
| **Status** | resolved |
| **Module** | BlueskyProfile |
| **Platform** | All |
| **First seen** | 2026-04-30 |

## Description

The profile screen has several visual bugs:

1. **No banner image** — the banner area at the top is blank/dark grey even when the account has a banner set on bsky.app.
2. **Avatar placeholder not themed** — the avatar shows a plain grey circle with the initial "B" rather than the styled `AvatarView` used elsewhere.
3. **Post cards have white background in dark mode** — the posts tab renders cards with a white background, the same issue seen in search results (#0051).

## Steps to reproduce

1. Navigate to the Profile tab.
2. Observe the banner area, avatar, and post list.

## Expected behavior

- Banner image loaded from the profile's `banner` URL.
- Avatar uses `AvatarView` with the standard blue gradient placeholder.
- Post cards respect dark mode via `.adaptiveBlueskyTheme()`.

## Actual behavior

Banner is blank. Avatar is an unstyled grey circle. Post cards are white in dark mode.

## Attachments

![Profile screen showing missing banner, plain avatar, and white post cards in dark mode](screenshot.png)

## Notes

`ProfileHeaderView` likely does not load or display the `banner` URL. The avatar rendering may be using a plain `Circle()` instead of `AvatarView`. The post list in `ProfileScreen` needs `.adaptiveBlueskyTheme()` applied to its `List` or `ScrollView`, same fix as #0051.
