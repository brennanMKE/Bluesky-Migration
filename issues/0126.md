## Description

`ListDetailScreen` has only Members and Feed tabs. The React Native reference adds an **About** tab with: full list description, creation date, creator, list URI (copyable), labels applied (for mod lists), and metadata. Without an About tab, list metadata is split awkwardly into the header and not all of it has a home.

## Steps to reproduce

1. Open any list.
2. Observe only Members and Feed tabs.

## Expected behavior

Add a third tab "About" containing the list's full description, creator info, member count, creation date, and copyable list URI. For moderation lists, also show the labels-applied set.

## Actual behavior

No About tab.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/ProfileList/AboutSection.tsx`.
- File to edit: `BlueskyKit/Sources/BlueskyLists/ListDetailScreen.swift`.
- Cross-cuts with #0125 — some of the metadata that lives in the header per RN may instead live in About; align with RN's exact split when implementing.

## Root cause

`ListDetailScreen` exposed only `Members` and `Feed` tabs. Long-form
metadata that didn't fit in the header (#0125) — full description,
creation date, copyable list URI, list-level labels — had no surface
in the SwiftUI client. RN packs everything into the header, but the
SwiftUI screen needed a dedicated "About" tab to host the overflow
without bloating the header card.

## Fix

- **Tab strip**: replaced the hard-coded two-tab `Picker` with a list
  derived from the loaded `ListView`'s purpose. Curate lists render
  `Posts / People / About` (matching RN's `sectionTitlesCurate =
  [Posts, People]` plus the new About tab). Moderation lists render
  `Members / About` — RN renders no tabs for mod lists, but SwiftUI
  needs the second slot to expose the metadata.
- **About tab content** (`ListDetailScreen.aboutTab(for:)`):
  - Full description (multi-line, `theme.colors.textPrimary`).
  - Creator row with avatar, optional display name, and tappable
    handle that fires `onProfileTap` (same callback the header uses).
  - Member count (mirrored from header for parity).
  - Creation date — uses `list.indexedAt` formatted with
    `.formatted(date: .long, time: .omitted)`.
  - Copyable list URI rendered in a monospaced font with a "Copy"
    affordance that reuses the existing `copyToClipboard` helper.
  - Mod-list labels: any `Label` records attached directly to the
    list view (`list.labels`) are rendered as capsule chips. Pure
    labeler-issued labels-applied are not exposed on `ListView` and
    are intentionally skipped (see Gotchas).
- **Reuse**: the About tab reads from the same `viewModel.list`
  value the header consumes; no new state, no new fetches.

## Files changed

- `BlueskyKit/Sources/BlueskyLists/ListDetailScreen.swift` — dynamic
  tab strip, `aboutTab(for:)` view, `aboutSection(title:content:)`
  helper.

## Gotchas

- **Labels-applied set is not exposed on `ListView`**. RN's
  `AboutSection` does not render this either — moderation lists
  publish their effective labels via separate labeler records on
  the labeler service, not as a property on
  `app.bsky.graph.defs#listView`. The About tab therefore renders
  only the labels attached to the list record itself (e.g. `!hide`).
  When the lexicon grows a labels-applied field on `ListView`,
  surface it in the existing `Labels` section.
- **Curate lists drop a tab in RN's spec**. RN curate = `[Posts,
  People]` (no About tab); SwiftUI now adds a third About tab. This
  is a deliberate divergence, called out in the issue spec, because
  the SwiftUI header is more compact than RN's
  `ProfileSubpageHeader` and needs an overflow surface.
- **`creator.createdAt`** was considered as a fallback for "Created"
  when `list.indexedAt` is missing, but `indexedAt` is the canonical
  list-creation timestamp in the lexicon. The "Created" row only
  renders when `indexedAt` is present.
