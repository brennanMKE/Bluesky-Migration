## Description

Manual sign-off for the Home feed against the React Native reference.
Covers feed switching, content rendering, infinite scroll, pull-to-
refresh, and the post action bar.

## Test sequence

1. **Default feed loads.** Launch the app signed in. Confirm the
   default feed (Following or Discover, depending on user prefs)
   renders within ~1 second.
2. **Feed selector — pinned feeds.** Tap/scroll to the feed selector
   chips (or saved-feeds toolbar item). Confirm all the user's
   pinned feeds appear in the same order RN shows.
3. **Switch feeds.** Tap a non-default feed. Confirm the feed switches,
   the scroll position resets to top, and the previous feed's items
   are unloaded (no ghost rows).
4. **Hide replies / hide reposts toggles.** Open the feed filter
   menu. Toggle "Hide replies" — confirm reply posts disappear
   without a re-fetch. Toggle "Hide reposts" — same.
5. **Per-feed filter state.** Switch to a different feed, then back.
   Confirm each feed remembers its own filter state independently.
6. **Pull to refresh.** Drag the list past the top edge. Confirm a
   spinner appears, the list re-fetches, and any new posts since
   the last fetch appear at the top.
7. **Infinite scroll.** Scroll past the bottom of the loaded posts.
   Confirm a spinner appears and the next page of posts loads.
8. **Post card content parity.** For a post with text + image embed +
   like/repost/reply counts, confirm:
   - avatar, display name, handle, relative timestamp present in the
     same positions as RN
   - text rendered with rich-text facets (mentions/links blue,
     hashtags as tags)
   - image embed rendered inline, tappable to fullscreen
   - action bar shows reply, repost, like, bookmark, share, and a
     `…` overflow menu — in that order (matches RN)
9. **Like / repost / bookmark interactions.** Tap each. Confirm the
   count increments instantly (optimistic), the icon swaps to the
   filled/coloured variant, and tapping again undoes the action.
10. **Share / copy link / open in browser.** Open the `…` menu on
    any post. Confirm at minimum: Copy link, Share via system sheet,
    Open in browser, Mute thread, Mute user, Block user, Report.
11. **Tap a post → thread.** Tap the post body. Confirm navigation
    to ThreadView with the focal post at top.
12. **Tap an author → profile.** Tap the avatar or display name.
    Confirm navigation to ProfileScreen for that user.
13. **Tap a hashtag → hashtag view.** Tap a `#hashtag` in post text.
    Confirm navigation to the hashtag screen.

## Expected behavior

Each step matches RN visually and behaviorally — same icons, same
ordering, same animation timing for likes/reposts, same overflow
menu items in the same order.
