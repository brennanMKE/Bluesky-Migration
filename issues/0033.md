## Live testing observations (2026-04-30)

**Messages tab fails to load** — the conversation list shows an error banner: "The operation couldn't be completed. (BlueskyCore.ATError error 2.)"

**Root cause identified from logs:**
```
convos fetch error: BlueskyCore.ATError.xrpc(code: "MethodNotImplemented", message: "Method Not Implemented")
request convos fetch error: BlueskyCore.ATError.xrpc(code: "MethodNotImplemented", message: "Method Not Implemented")
```

The `chat.bsky.convo.listConvos` calls are being sent to `bsky.social`, but the Bluesky chat service runs on a separate proxy endpoint (`https://api.bsky.chat`). All `chat.bsky.*` lexicons must be routed to this endpoint instead of the standard PDS. The `ATProtoClient` needs to detect `chat.bsky.*` lexicons and use the chat service URL.

## Incomplete reason

Validation gate — requires routing `chat.bsky.*` requests to `https://api.bsky.chat` before the conversations list will load. Cannot be validated until that fix is in place.

## Root cause

The Bluesky chat service is a separate AT Protocol service from the user's PDS. The PDS does not implement `chat.bsky.*` itself — those XRPC calls must be proxied to `did:web:api.bsky.chat#bsky_chat` via the `atproto-proxy` request header. Without that header the PDS returned `MethodNotImplemented`, and `ConversationStore.loadInitial` surfaced the resulting `ATError.xrpc` as the "BlueskyCore.ATError error 2" banner on the Messages tab.

The React Native reference handles this with `DM_SERVICE_HEADERS` in `src/lib/constants.ts`, applied to every `agent.chat.bsky.*` call. The Swift port had the lexicon types and store calls in place but never set the proxy header.

## Fix

Added an `applyProxyHeader(_:lexicon:)` helper to `ATProtoClient` and called it from `buildGetRequest`, `buildPostRequest`, and `buildUploadRequest`. When a lexicon starts with `chat.bsky.`, it sets `atproto-proxy: did:web:api.bsky.chat#bsky_chat` on the outgoing request. The request URL is unchanged — the user's PDS still receives the request and forwards it to the chat service based on the header. The helper is structured so additional proxied services (labeler, push notifications) can be added the same way later.

- `BlueskyKit/Sources/BlueskyNetworking/ATProtoClient.swift` — added `applyProxyHeader` and three call sites in the request builders.
