## Description

The React Native Settings hub includes a **Help** row between Languages and About that opens the Bluesky help desk in the system browser (`https://blueskyweb.zendesk.com/` or whatever `HELP_DESK_URL` resolves to). The SwiftUI `SettingsScreen` has no Help row at all.

## Steps to reproduce

1. Open Settings.
2. Look for a "Help" row.
3. Observe there is none.

## Expected behavior

Add a "Help" row to `SettingsScreen` between Languages and About sections. Tapping opens the help desk URL via `openURL`.

## Actual behavior

No Help row.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Settings/Settings.tsx` (line 245-254 — `PressableItem` with `CircleQuestionIcon`, opens `HELP_DESK_URL` via `Linking.openURL`).
- `HELP_DESK_URL` source: `Bluesky-ReactNative/src/lib/constants.ts` (line 15) — resolves to `https://blueskyweb.zendesk.com/hc/en-us`.
- File edited: `BlueskyKit/Sources/BlueskySettings/SettingsScreen.swift`.

## Root cause

`SettingsScreen.swift` had no row that links to the Bluesky help desk. RN's `Settings.tsx` ships a `Help` row immediately above `About` that opens `HELP_DESK_URL` in the system browser; the SwiftUI port skipped it.

## Fix

Added a `Help` row to the bottom-most section of the Settings hub, placed directly above `About` to match RN's adjacency. The row is a `Button` (RN uses `PressableItem`, not `LinkItem`) with the `questionmark.circle` SF Symbol (RN uses `CircleQuestionIcon`) and an accessibility hint mirroring RN's `Opens helpdesk in browser`. Tapping calls `openURL` with a file-private `helpDeskURL` constant equal to RN's `HELP_DESK_URL` (`https://blueskyweb.zendesk.com/hc/en-us`). The constant is kept private to the file since it has no other consumer; promoting it to a shared module would be premature.

## Files changed

- `BlueskyKit/Sources/BlueskySettings/SettingsScreen.swift` — added file-private `helpDeskURL`, an `@Environment(\.openURL)` property, and the `Help` row above `About`.

## Gotchas

- RN's settings hub is one flat list; the SwiftUI port uses `Section` groups. The `About` row already lives in its own unnamed trailing section, so `Help` was added to that same section to keep RN's "Help directly above About" adjacency without inventing a new visual grouping.
- RN's row is a `PressableItem` (action), not a `LinkItem` (push navigation), because the destination is an external URL. Mirrored that with `Button { openURL(...) }` rather than `NavigationLink`.
