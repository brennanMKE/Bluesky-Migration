# 0051 — Search results post cards have white background in dark mode

| | |
|---|---|
| **Status** | in-progress |
| **Module** | BlueskySearch |
| **Platform** | All |
| **First seen** | 2026-04-29 |

## Description

Post cards in search results (hashtag feed, post search results) render with a white/light background even when the app is in dark mode. The surrounding chrome (sidebar, navigation bar) is dark, but each post card has an unthemed white background.

## Steps to reproduce

1. Run the app in dark mode.
2. Go to the Search tab and search for a hashtag or keyword.
3. Observe the post results list.

## Expected behavior

Post cards use the dark theme background color, matching the rest of the app.

## Actual behavior

Post cards have a white background, creating a jarring light-on-dark contrast.

![Search results showing white post card backgrounds in dark mode](screenshot.png)

## Notes

`HashtagView` and `SearchScreen` render `PostCard` but likely do not apply `.adaptiveBlueskyTheme()` or `.blueskyTheme(.dark)` to the list container. `PostCard` reads `theme.colors.background` from the environment — if no theme is injected, it falls back to the default (light) theme. The fix is to add `.adaptiveBlueskyTheme()` to the scroll/list views in `HashtagView` and the post-results section of `SearchScreen`.
