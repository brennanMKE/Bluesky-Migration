# 0066 — Module 15 gate (iOS): remaining screens feature parity on iPhone

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyLists / BlueskyFeed / BlueskyModeration / BlueskySettings |
| **Platform** | iOS |
| **First seen** | 2026-05-05 |

## Description

Once the iOS build is restored (#0061), the Module 15 feature-parity gate must be re-run on a real iPhone. Most screens are platform-agnostic, but several have iOS-specific behavior that cannot be exercised on macOS — most notably the Video Feed (TikTok-style full-screen vertical scroll) and any screens that rely on iOS swipe gestures, the share sheet, or the system PHPicker.

This issue split off from #0038, which now focuses on the macOS gate.

## Steps to reproduce

1. Resolve #0061 so the iOS build is green.
2. Install the app on a registered iPhone (or run in iOS Simulator) and sign in.
3. Run the live checks below.

## Live checks (must all pass)

- **Lists** — Create a list; add a member; open the list feed. Confirm the iOS swipe-to-edit / swipe-to-delete on member rows behaves as expected.
- **Starter Packs** — Open a starter pack; view members; tap Follow All; confirm haptic feedback (if any) fires and follow relationships are created.
- **Saved Feeds** — Pin/unpin a feed; reorder via the iOS drag-handle / long-press; force-quit and relaunch; confirm order persists.
- **Video Feed** — Open the vertical Video Feed. Scroll several clips with the iOS rubber-band swipe; confirm clips auto-play with sound; tap to mute/unmute; confirm pagination loads more clips on scroll. Background and return to foreground; confirm playback resumes correctly.
- **Labeler Profile** — Open a labeler; subscribe; back-swipe out; reopen; confirm subscription state restored from server.
- **App Passwords** — Create an app password; tap-and-hold or long-press to copy the token (iOS clipboard semantics); revoke an existing password; confirm list refreshes.
- **Bookmarks** — Bookmark a post; open Bookmarks; confirm post appears; swipe-to-delete via iOS swipe action; confirm post disappears.
- **Share sheet integration** — On any post or list, tap Share; confirm the iOS share sheet (`UIActivityViewController`) appears with valid URL/text; share to a Notes pin; confirm the destination receives the right content.
- **Pull-to-refresh** — On Bookmarks, Saved Feeds, and Lists, pull down via the iOS rubber-band gesture; confirm refresh fires and indicator dismisses correctly.

## Expected behavior

All nine checks pass. iOS-specific surfaces (vertical video, swipe gestures, share sheet, rubber-band pull-to-refresh) behave correctly without regressions from the macOS-focused work.

## Actual behavior

Cannot be validated until #0061 is resolved.

## Notes

- Video Feed is the highest-risk surface — its UX paradigm (full-screen vertical, auto-play, swipe-driven pagination) is iOS-native and may need iOS-specific code that was deprioritized while macOS was the primary target. Confirm before closing whether the macOS rendering used in #0038 is even meant to ship as the same screen, or whether iOS should have a distinct implementation.
