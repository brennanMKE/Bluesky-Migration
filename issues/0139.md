## Description

In the React Native client, tapping the like, repost, or quote count on a focal post in `ThreadView` opens a paginated list of the corresponding actors / posts:

- **Liked by** — `app.bsky.feed.getLikes` → list of profiles who liked.
- **Reposted by** — `app.bsky.feed.getRepostedBy` → list of profiles who reposted.
- **Quotes of** — `app.bsky.feed.getQuotes` → feed of posts quoting this one.

The SwiftUI thread view shows the counts (or doesn't, in the focal-post case) but the counts are not tappable and the destination screens don't exist.

## Steps to reproduce

1. Open any thread on a popular post.
2. Tap on the "X likes" / "Y reposts" stat row above the action bar.
3. Observe nothing happens.

## Expected behavior

- The focal post (and any post when its detail is opened) shows a tappable stat row: "X likes · Y reposts · Z quotes".
- Tapping each navigates to a dedicated screen:
  - `LikedByScreen` — paginated profile list.
  - `RepostedByScreen` — paginated profile list.
  - `QuotesOfScreen` — paginated post feed (uses `PostCard` rows).
- Each screen supports infinite scroll and pull-to-refresh.

## Actual behavior

Counts are not interactive; the destination screens don't exist.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/PostLikedBy.tsx`, `PostRepostedBy.tsx`, `PostQuotes.tsx`.
- AT Proto endpoints: `app.bsky.feed.getLikes`, `getRepostedBy`, `getQuotes`. Confirm `BlueskyCore`/`BlueskyNetworking` types exist; if not, add.
- Files to create: `BlueskyKit/Sources/BlueskyFeed/LikedByScreen.swift`, `RepostedByScreen.swift`, `QuotesOfScreen.swift`. Wire from `ThreadView`'s focal post stat row.
- Pairs with #0148 (focal post emphasis) which is what makes the stat row visible in the first place.

## Root cause

The Liked By / Reposted By / Quotes Of destinations had never been ported from RN. `BlueskyCore` did not model the response payloads for `app.bsky.feed.getLikes`, `getRepostedBy`, or `getQuotes`, and no `BlueskyFeed` screen existed to render them. Without callbacks on `ThreadView`, even a future stat row had no destination to push to.

## Fix

Added the three lexicon response types in `BlueskyCore/Feed.swift`:

- `GetLikesResponse` (with `LikeActor` for the `likes[]` element).
- `GetRepostedByResponse` (`repostedBy: [ProfileView]`).
- `GetQuotesResponse` (`posts: [PostView]`).

Created three screens in `BlueskyKit/Sources/BlueskyFeed/`:

- `LikedByScreen` + `LikedByViewModel` — `@Observable @MainActor`; paginated `List` of profile rows (avatar + display name + handle + bio + verified badge), 30 per page, infinite-scroll on tail row appearance, pull-to-refresh.
- `RepostedByScreen` + `RepostedByViewModel` — same pattern over `repostedBy`.
- `QuotesOfScreen` + `QuotesViewModel` — paginated `LazyVStack` of `PostCard`s with the same infinite-scroll/refresh wiring. Mirrors RN's `usePostQuotesQuery` `select` step by stripping posts whose embedded record decodes as `app.bsky.embed.record#viewDetached` (lands in `EmbedRecordContent.unknown` since the lexicon discriminator isn't enumerated yet).

Wired three optional callbacks on `ThreadView` — `onLikedByTap`, `onRepostedByTap`, `onQuotesTap` — taking an `ATURI`. The stat-row UI that fires them is #0146's responsibility; the callbacks already exist so #0146 can land without follow-on plumbing. The recursive `selectedReplyURI` destination forwards them to the inner `ThreadView` so deeper navigation keeps the same hooks.

## Files changed

- `BlueskyKit/Sources/BlueskyCore/Feed.swift` — added `LikeActor`, `GetLikesResponse`, `GetRepostedByResponse`, `GetQuotesResponse`.
- `BlueskyKit/Sources/BlueskyFeed/LikedByScreen.swift` — new.
- `BlueskyKit/Sources/BlueskyFeed/RepostedByScreen.swift` — new.
- `BlueskyKit/Sources/BlueskyFeed/QuotesOfScreen.swift` — new.
- `BlueskyKit/Sources/BlueskyFeed/ThreadView.swift` — added the three optional tap callbacks.

## Gotchas

- `VerifiedBadge.forProfile(_:)` is typed against `ProfileBasic`, not `ProfileView`, so the rows access `verification?.isVerified` directly instead of going through the convenience.
- The detached-quote filter relies on the `viewDetached` `$type` falling into `EmbedRecordContent.unknown(...)`. If that variant ever gets a first-class case in `BlueskyCore/Embed.swift`, update `QuotesViewModel.filterDetached` to match it explicitly.
- The screens are reachable today only if a caller wires the callbacks. The stat-row UI in `ThreadView`'s focal post (issue #0146) is the next step — at which point `MainTabView` should attach `navigationDestination(item:)` blocks for each callback so the parent owns the push, keeping `BlueskyFeed` free of an upward import.
