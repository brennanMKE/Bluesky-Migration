# 0139 — Liked-by, Reposted-by, and Quotes-of screens are missing

| | |
|---|---|

## Description

In the React Native client, tapping the like, repost, or quote count on a focal post in `ThreadView` opens a paginated list of the corresponding actors / posts:

- **Liked by** — `app.bsky.feed.getLikes` → list of profiles who liked.
- **Reposted by** — `app.bsky.feed.getRepostedBy` → list of profiles who reposted.
- **Quotes of** — `app.bsky.feed.getQuotes` → feed of posts quoting this one.

The SwiftUI thread view shows the counts (or doesn't, in the focal-post case) but the counts are not tappable and the destination screens don't exist.

## Steps to reproduce

1. Open any thread on a popular post.
2. Tap on the "X likes" / "Y reposts" stat row above the action bar.
3. Observe nothing happens.

## Expected behavior

- The focal post (and any post when its detail is opened) shows a tappable stat row: "X likes · Y reposts · Z quotes".
- Tapping each navigates to a dedicated screen:
  - `LikedByScreen` — paginated profile list.
  - `RepostedByScreen` — paginated profile list.
  - `QuotesOfScreen` — paginated post feed (uses `PostCard` rows).
- Each screen supports infinite scroll and pull-to-refresh.

## Actual behavior

Counts are not interactive; the destination screens don't exist.
