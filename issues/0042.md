## Description

The list/feed-switcher button (≡) in the Home screen navigation bar is positioned in the center of the toolbar rather than being grouped with the other trailing toolbar buttons (compose and >>). This leaves an odd gap and breaks the expected right-aligned button cluster.

## Steps to reproduce

1. Launch the app on macOS.
2. Navigate to the Home tab.
3. Observe the navigation bar — the ≡ button sits in the center, while the compose and >> buttons are on the right.

## Expected behavior

All three toolbar buttons (≡, compose, >>) should be right-aligned as a group in the trailing position of the navigation bar.

## Actual behavior

The ≡ button is centered in the toolbar; the compose and >> buttons are right-aligned. They are not grouped together.

## Notes

Likely a `toolbar` placement issue in the SwiftUI navigation view — the ≡ button is probably placed with `.principal` or no explicit placement instead of `.navigationBarTrailing` / `.topBarTrailing`. Check the toolbar modifier in the Home/Feed screen's navigation stack.

## Root cause

`FeedView` and `MainTabView` each declared their own `.toolbar` block: `FeedView` added the filter button (`line.3.horizontal.decrease.circle`) with `.primaryAction`, and `MainTabView` (the parent) added compose (`square.and.pencil`) and saved-feeds (`list.star`) with `.primaryAction`. On macOS NavigationSplitView, when toolbar items declared in nested `.toolbar` modifiers all use `.primaryAction`, SwiftUI assigns the outer modifier's items to the trailing region and pushes inner-modifier items to `.principal` (centered). The earlier fix (commit `06c1bec`) only changed the saved-feeds placement and didn't address the multi-modifier merge, so the filter button stayed centered.

## Fix

Removed `FeedView`'s `.toolbar` modifier entirely and rendered the filter Menu inline beside the `FeedSwitcherView` row (Following / Discover tabs) using `.menuStyle(.borderlessButton)` and `.menuIndicator(.hidden)`. `MainTabView` is now the only view contributing items to the Home window toolbar, so compose and saved-feeds stay grouped on the trailing edge.

## Files changed

- `BlueskyKit/Sources/BlueskyFeed/FeedView.swift` — moved filter Menu from `.toolbar` into the body, in an HStack with `FeedSwitcherView`.

## Gotchas

- The filter button is no longer part of the macOS title bar. It now sits at the trailing end of the Following / Discover tab row. This is the trade-off for keeping the toolbar items grouped on the right.
- `FeedView` still owns its filter `@State` so the public initializer is unchanged — `MainTabView` did not need updating.
