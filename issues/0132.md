# 0132 — Muted Words & Tags screen is missing

|---|---|

## Description

The React Native moderation hub includes a **Muted Words & Tags** entry that opens a screen (or dialog) where users curate a list of strings to filter from feeds. Each entry can target tag-only or text-anywhere matches, and can target specific timeframes (forever / 24h / 7d / 30d) and contexts (content vs tags). The SwiftUI moderation surface has no equivalent — there is no way to mute a specific word, hashtag, or domain.

This is one of the most-used moderation tools in RN and its absence is a significant gap.

## Steps to reproduce

1. Open the SwiftUI Moderation hub.
2. Look for "Muted Words" / "Muted Tags".
3. Observe neither row exists.

## Expected behavior

A new "Muted words & tags" row in the Moderation hub. Tapping opens `MutedWordsScreen.swift`:

- Add new entry: text field, target picker (Content + tags / Tags only / Content only), duration picker (Forever / 24h / 7d / 30d).
- List of existing entries with edit and delete actions.
- Persists via `app.bsky.actor.putPreferences` under `app.bsky.actor.defs#mutedWordsPref`.
- Feed-side filtering: `FeedViewModel` (or a higher-up filter) drops posts matching any non-expired muted word.

## Actual behavior

No screen, no preference handling, no filtering.
