## Description

Manual sign-off for ProfileScreen — the user's own profile and other
users' profiles. Covers header (banner, avatar, name, handle, bio,
counts, badges), tabs (Posts / Replies / Media / Feeds / Lists /
Starter Packs), and the follow / block / mute / report flows.

## Test sequence — own profile

1. **Open own profile.** Tap the Profile tab. Confirm the screen
   loads with no spinner if cached.
2. **Header parity.**
   - Banner image fills the top, edge to edge.
   - Avatar overlaps the bottom of the banner.
   - Display name + handle stacked under the avatar.
   - Bio below.
   - Followers / Following / Posts counts in a row beneath the bio,
     each tappable.
   - "Edit Profile" button is primary-styled and right-aligned.
3. **Edit profile.** Tap Edit Profile. Confirm sheet with avatar
   picker, banner picker, display name field, bio field. Edit
   bio, save. Confirm the change is visible on profile and other
   surfaces.
4. **Verified / labeler badges.** If the account has verification or
   labels, confirm badges render at the correct positions (#0023).
5. **Known followers chip.** If applicable, confirm the chip appears
   under the header (#0024) — does not apply to own profile.
6. **Tabs row.** Confirm five tabs in this order: Posts, Replies,
   Media, Feeds, Lists, Starter Packs. Confirm each tab loads
   content and is independently scrollable.
7. **`…` menu.** Confirm Settings / Bookmarks / My Lists / Moderation
   entries are present (already covered by app-shell menu).

## Test sequence — other user's profile

8. **Open from feed.** Tap an author in the feed. Confirm
   ProfileScreen for that account loads.
9. **Header parity.** Same as above, except the primary button is
   "Follow" / "Following" (toggles) and shield-shaped overflow shows
   Mute / Block / Report.
10. **Follow / unfollow.** Tap Follow. Confirm the button label
    toggles to "Following" and the followers count increments by 1
    optimistically. Tap again, confirm unfollow.
11. **Block.** Open the overflow → Block. Confirm a confirmation
    sheet (matches RN), then the block applies, the timeline goes
    empty for this profile, and the user is added to mutual blocks
    on the next refresh of moderation lists.
12. **Mute.** Open the overflow → Mute. Confirm the mute applies
    and posts from this user disappear from the feed on next refresh.
13. **Report.** Open the overflow → Report. Confirm a categorized
    picker (matches RN), allows selecting a category + labeler, and
    submitting closes the sheet with a confirmation toast (#0135).
14. **Known followers chip.** Confirm the chip displays mutual
    followers — first 3 avatars + "Followed by X and Y" (#0024).
15. **Tabs on another profile.** Confirm all six tabs load that
    user's content correctly.

## Sign-off checklist

### Own profile

- [ ] Header (banner / avatar / counts / Edit) matches RN
- [ ] Edit profile sheet saves and reflects changes
- [ ] All six tabs load and scroll independently

### Other users

- [ ] Header layout matches RN for non-self
- [ ] Follow / unfollow is optimistic and stable
- [ ] Block / mute / report flows are RN-parity
- [ ] Known followers chip renders correctly
- [ ] Verified / labeler badges render at correct positions
- [ ] No regressions in #0023, #0024, #0025, #0056, #0155, #0159
