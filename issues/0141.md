# 0141 — Thread view doesn't render nested reply tree (no indentation, no connectors, no "show more replies")

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyFeed |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

`ThreadView` flattens replies with `flattenThread()` and renders them in a single column at uniform indentation. The React Native client renders a tree:

- **Indentation** per nesting level (RN uses a `TREE_INDENT` constant).
- **Vertical connector lines** on the left edge of each row, joining a reply to its parent above.
- **"Show N more replies"** expanders for collapsed branches deeper than the default depth.
- **Avatar clustering**: the thinner left rail uses smaller avatars when nested.

Without this rendering, multi-branch threads collapse into a flat sequence and the conversation structure is lost.

## Steps to reproduce

1. Open a thread with replies that themselves have replies.
2. Compare to the same thread in RN.
3. Observe SwiftUI shows everything flat.

## Expected behavior

`ThreadView` recursively renders the reply tree:

- Each row's left edge has a vertical connector line drawn from its parent's row, indented by depth × `TREE_INDENT`.
- Branches deeper than ~2 levels collapse to a "Show N more replies" button; tapping expands inline.
- Indentation caps at a sensible max (RN uses ~6 levels) before flattening to keep readability on narrow screens.

## Actual behavior

`flattenThread()` returns a flat array; `ForEach` renders rows with no indentation, no connectors, no expanders.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/com/post-thread/ThreadItemTreePost.tsx` and the line-drawing helpers.
- File to edit: `BlueskyKit/Sources/BlueskyFeed/ThreadView.swift` — keep recursion via `AnyView` (already used to break self-reference). Add a `depth: Int` parameter to the recursive renderer and use it to drive horizontal padding + connector overlay.
- Connector lines: a `Rectangle().frame(width: 2)` overlay at the leading edge with absolute positioning works for SwiftUI; alternative is a custom `Shape`.
