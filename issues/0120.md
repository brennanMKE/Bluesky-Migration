# 0120 — Language settings missing app language and content-language multi-select

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskySettings |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native Language Settings screen has three distinct controls:

1. **App language** — a single-select picker for the UI's display language (locale), applied via the i18n layer.
2. **Primary post language** — single-select; the default `langs[]` value attached to new posts.
3. **Content languages** — **multi-select** of which languages the user wants to see in feeds; posts in unselected languages are filtered.

The SwiftUI `LanguageSettingsScreen` only shows a hardcoded single-select list of 12 languages used as Post Languages. App language and content-language multi-select are missing entirely.

## Steps to reproduce

1. Settings → Language.
2. Compare to RN — SwiftUI shows one list; RN shows three sections.

## Expected behavior

Refactor `LanguageSettingsScreen` into three sections:

- **App language** — picker bound to `Locale` override (applied via `.locale(...)` modifier or `Bundle.preferredLocalizations`).
- **Primary post language** — picker; this becomes the default `langs[0]` for new posts.
- **Content languages** — multi-select with checkboxes; consumed by feed-side filtering.

Persist via `UserDefaultsPreferencesStore` (locally) and `app.bsky.actor.putPreferences` (server-side, under `app.bsky.actor.defs#contentLanguagesPref`).

## Actual behavior

Single hardcoded post-languages list, no app-language picker, no content-language multi-select.

- Lexicon: `app.bsky.actor.defs#contentLanguagesPref` and `#postLabelsPref`.
