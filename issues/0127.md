# 0127 — Lists have no edit or delete UI

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyLists |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

Once a list is created in the SwiftUI app, it cannot be edited or deleted. The React Native client lets the list creator:

- **Edit** the list — change name, description, avatar, purpose. Reuses the create form in edit mode.
- **Delete** the list — destructive confirmation, then `com.atproto.repo.deleteRecord` on `app.bsky.graph.list`.
- **Add / remove members** — managed inline from the Members tab.

SwiftUI has `ListCreateSheet` for creation but no edit path and no delete affordance.

## Steps to reproduce

1. Create a list.
2. Open it.
3. Look for an Edit button or a delete option in any menu.
4. Observe none exist.

## Expected behavior

In the list-detail header's secondary-actions menu (added by #0125):

- **Edit list** — opens an edit sheet that mirrors `ListCreateSheet` but pre-populated and writes via `com.atproto.repo.putRecord`.
- **Delete list** — destructive confirm dialog. On confirm, deletes the list record and pops to `ListsScreen`.

The Members tab gains Add / Remove member actions for the list creator.

## Actual behavior

No edit or delete UI; lists are append-only after creation.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/ProfileList/Header.tsx` (menu) and `src/screens/Lists/components/EditListDialog.tsx`.
- Endpoints: `com.atproto.repo.putRecord` (edit), `com.atproto.repo.deleteRecord` (delete), `com.atproto.repo.applyWrites` (member add/remove batches).
- Files to edit: `BlueskyKit/Sources/BlueskyLists/ListDetailScreen.swift` (menu wiring), `ListsViewModel.swift` / `ListDetailViewModel.swift` (mutations), and a new `ListEditSheet.swift`.
