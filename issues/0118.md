# 0118 — External Media Preferences screen is missing (per-source toggles for YouTube, Spotify, etc.)

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskySettings |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native client has an **External Media Preferences** screen (Settings → Content & Media → External media) with per-source toggles for which embedded players to inline-render: YouTube, YouTube Music, Spotify, Apple Music, Twitch, SoundCloud, Vimeo, Giphy, Tenor, Flickr, etc. Sources that are off render as plain link cards instead of inline players.

The SwiftUI `ContentSettingsScreen` has only a single boolean "Load external embeds" toggle. The per-source granularity is lost; users either get all third-party iframes or none.

## Steps to reproduce

1. Settings → Content & Media.
2. Look for per-source toggles.
3. Observe a single coarse toggle.

## Expected behavior

A new `ExternalMediaPreferencesScreen.swift` accessible from `ContentSettingsScreen` (or replace the single toggle with a navigation row). Each source row has its own toggle. Defaults match RN. Embed renderers in `BlueskyUI` (specifically `PostEmbedView`) consult the per-source flags before rendering inline players.

## Actual behavior

Single toggle; no per-source granularity.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Settings/ExternalMediaPreferences/index.tsx`.
- Storage: RN keeps these in local preferences (not in AT Proto preferences) — verify the same is appropriate here. They probably belong in `UserDefaultsPreferencesStore`.
- Files to create: `BlueskyKit/Sources/BlueskySettings/ExternalMediaPreferencesScreen.swift`, plus a small struct in `BlueskyKit` for the per-source flags so feeds can read them.
