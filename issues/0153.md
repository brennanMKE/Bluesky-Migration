# 0153 — App icon asset catalog is empty; populate from `AppIcon.png` for iOS / iPadOS / macOS

| | |
|---|---|
| **Status** | open |

## Description

`Bluesky-SwiftUI/Bluesky-SwiftUI/Assets.xcassets/AppIcon.appiconset/` exists but contains only a `Contents.json` with **macOS-only** size declarations and **no image files** referenced. The result: the app ships with a generic placeholder icon, no iOS / iPadOS coverage, and no chance for the home-screen / Dock / Finder to render the proper Bluesky butterfly icon.

A 1024×1024 source `AppIcon.png` has been placed at `/Users/brennan/Developer/ReactNative/Bluesky-SwiftUI/AppIcon.png` (verified: 1024×1024 RGBA, 18 KB) ready to be processed.

## Steps to reproduce

1. Open `Bluesky.xcworkspace`.
2. Build and run the app on any platform.
3. Look at the icon in the Dock (macOS) or home screen (iOS / iPadOS).
4. Observe the generic placeholder.

## Expected behavior

A populated `AppIcon.appiconset` with the full size matrix for iOS, iPadOS, and macOS, generated from `AppIcon.png`. After build, the app shows the Bluesky icon everywhere the system renders an app icon.

## Actual behavior

`AppIcon.appiconset/Contents.json` declares macOS slots only and references no `filename` for any of them; iOS / iPadOS slots are missing entirely.

## Implementation

The `icongen` CLI (already on the developer's `$PATH`) produces a complete `AppIcon.appiconset` for the requested platforms from a 1024×1024 source. Usage:

```
icongen -h
USAGE: icongen [-p <platforms>] [-o <output>] [tvos-flags] <source.png>
ARGUMENTS:
  <source.png>     Path to 1024x1024 source PNG (required)
OPTIONS:
  -p <platforms>   Comma-separated platforms: iOS,iPadOS,macOS,tvOS or "all"
  -o <output>      Output directory (default: ./Assets.xcassets)
```

Concrete command for this project:

```
cd /Users/brennan/Developer/ReactNative/Bluesky-SwiftUI && \
icongen -p iOS,iPadOS,macOS \
  -o Bluesky-SwiftUI/Assets.xcassets \
  AppIcon.png
```

Steps to land:

1. Run the command above; it overwrites `Bluesky-SwiftUI/Assets.xcassets/AppIcon.appiconset/` with the full iOS + iPadOS + macOS matrix and the per-size scaled PNGs.
2. Verify by opening `Bluesky.xcworkspace` and inspecting the asset catalog — every platform's slots should show the icon thumbnail.
3. Build for each target (`xcodebuild -destination 'platform=macOS' build`, plus iOS Simulator) and confirm the icon renders in the Dock / Springboard.
4. Don't ship the `AppIcon.png` source into the project; it stays at the repo root as the editable master. Optionally `.gitignore` the working file if it isn't tracked yet — confirm before adding to a `.gitignore`.

## Notes

- `icongen` is the Brennan tooling; same pattern used elsewhere. Don't hand-author `Contents.json` — the matrix changes between Xcode versions and `icongen` keeps it current.
- The current `Contents.json` uses the legacy per-size `appiconset` format. That's fine for this project's deployment baseline; the newer single-1024 unified format (Xcode 14+) is also acceptable but `icongen` produces the per-size matrix, which works on every Xcode version.
- For full RN parity with the alternate icons (Aurora, Bonfire, Classic, Flat Black/Blue/White, Midnight, Sunrise, Sunset, Legacy Dark/Light), see the source assets at `Bluesky-ReactNative/assets/app-icons/ios_icon_core_*.png`. Wiring `setAlternateIconName(_:)` and a settings UI is a separate follow-up — out of scope for this issue.
- The `xcode-app-icons` skill is available if generating manually instead of via `icongen` ever becomes necessary.
