# 0104 — Conversation list row missing timestamp, system-message handling, and multi-line preview

|---|---|

## Description

The SwiftUI `ConvoRow` (in `ConversationListScreen.swift` ~lines 168–179) shows the avatar, display name, and a single-line of `message.text`. The React Native reference shows several more details:

- **Relative timestamp** right-aligned next to the unread badge (e.g. "2h").
- **System message handling**: when the last "message" is actually a system event ("Conversation deleted", "@alice left the group", "@bob added @carol"), RN renders an italicized system text instead of `message.text`.
- **Two-line preview** with truncation, not single-line. Long messages or multi-line content are properly handled.
- **Sender prefix on group chats**: e.g. "@alice: hello" so the row identifies who sent the last message in a group thread.

## Steps to reproduce

1. Open the SwiftUI Messages tab.
2. Compare a row to the RN reference: timestamp, multi-line preview, sender prefix on groups.

## Expected behavior

`ConvoRow` renders:

- Avatar (existing).
- Display name + handle (existing).
- Right-aligned relative timestamp ("now", "2h", "Yesterday", "Mar 4") matching RN's helper.
- 2-line preview with `lineLimit(2)` and truncation tail.
- For group convos: "@<sender>: <text>" prefix on the preview.
- For system events: italicized system text in place of the preview.
- Unread badge stays at trailing edge below the timestamp.

## Actual behavior

Single-line preview, no timestamp, no system handling, no sender prefix.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Messages/components/ChatListItem.tsx`.
- File to edit: `BlueskyKit/Sources/BlueskyMessages/ConversationListScreen.swift`.
- The system-message detection lives on the lexicon: `chat.bsky.convo.defs#deletedMessageView` and similar variants. `ConvoView.lastMessage` is a discriminated union — the SwiftUI side may need to handle the system-variant case in `ConversationStore.parseLastMessage`.
