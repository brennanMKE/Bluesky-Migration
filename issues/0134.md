## Description

`MutesScreen` and `BlocksScreen` render flat paginated lists with no search affordance. The React Native equivalents (built on `ProfileCard`) include a search field at the top so users with long mute/block lists can find and unmute/unblock a specific account quickly.

## Steps to reproduce

1. Mute or block enough accounts that the list grows beyond a screen.
2. Open Mutes (or Blocks).
3. Look for a search field.
4. Observe none.

## Expected behavior

Both screens get a `.searchable(text:)` modifier that filters the rendered profiles by handle/display-name match. Filtering is local (the loaded paged accounts) and the search returns to full list when cleared.

## Actual behavior

No search.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Moderation/MutedAccounts.tsx` / `BlockedAccounts.tsx`.
- Files to edit: `BlueskyKit/Sources/BlueskyModeration/MutesScreen.swift` and `BlocksScreen.swift`.

## Root cause

Both screens were built as plain `List`s over the loaded paged accounts, with no search field. RN's equivalent screens (built on `ProfileCard`) expose a top-of-list search input so long lists are navigable.

## Fix

- Added a `@State searchText` to each screen and wired `.searchable(text:, prompt: "Search…")` to the `List`.
- Added a `filteredMutes` / `filteredBlocks` computed property that lower-cases the trimmed query and substring-matches both `handle.rawValue` and `displayName`.
- When `searchText` is empty, the filtered list returns the full underlying paged list — preserving prior behavior.
- The pagination trigger (`onAppear` last-row check) continues to use `viewModel.mutes.last` / `viewModel.blocks.last` rather than the filtered tail, so paging keeps loading the next unfiltered page even while filtering is active.

## Files changed

- `BlueskyKit/Sources/BlueskyModeration/MutesScreen.swift`
- `BlueskyKit/Sources/BlueskyModeration/BlocksScreen.swift`

## Gotchas

- Pagination is intentionally tied to the unfiltered tail; binding it to the filtered list would stall paging the moment a query hides the real last row, hiding all unloaded accounts behind the filter.
- Filtering operates only on the loaded pages — searching for an account that is past the current pagination cursor will not surface it until subsequent pages load. RN behaves the same way.
