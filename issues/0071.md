# 0071 — iOS bottom chrome: redesign tab bar and add floating compose button

| | |
|---|---|
| **Status** | open |

| **First seen** | 2026-05-05 |

## Description

The iOS bottom chrome (tab bar + compose entry) needs to match the React Native reference. Today the iOS app uses a labeled five-tab bar with a generic "person" icon for the Profile tab and no floating compose button — the compose action lives in the top-right of every screen. RN ships an icon-only tab bar, uses the user's avatar for the Profile tab, indicates selection by swapping outline icons for filled ones, and surfaces a primary blue floating compose button (FAB) at bottom-right that overlaps the feed.

This is part of the larger iOS parity audit in #0068.

## Scope

### Tab bar

- **Drop the text labels.** Tabs render icon-only on iOS. (macOS keeps labels — this is iOS-only.)
- **Replace the Profile tab icon with the user's avatar.** A circular AsyncImage backed by the signed-in account's avatar URL, falling back to a placeholder if not loaded. Should respect dark/light backgrounds.
- **Swap selected-state.** Today the selected tab gets a pill-shaped capsule background and the icon stays the same. Match RN: the icon itself swaps from outline to filled when selected (e.g. `house` ↔ `house.fill`, `magnifyingglass` ↔ `magnifyingglass.circle.fill`, `bell` ↔ `bell.fill`). For Profile, ring the avatar with the brand color when selected.
- **Tab order stays Home · Search · Messages · Notifications · Profile** — that already matches.

### Floating compose button (FAB)

- **Add a circular blue button at bottom-right** with the pencil/compose SF Symbol, placed above the bottom tab bar with a 16pt margin from the right edge and ~16pt above the tab bar.
- **Tap opens the existing composer sheet** (`ComposerSheet`).
- **Visible on Home, Search, Notifications, Profile** (matches RN). Hide on Messages.
- **Removes the top-right compose action from those screens on iOS** — the FAB replaces it. (macOS keeps the top-right compose entry; this is iOS-only.)

## Implementation notes

- Code: `Bluesky-SwiftUI/Bluesky-SwiftUI/MainTabView.swift` is the natural place for the tab bar redesign and FAB overlay. Use `#if os(iOS)` to gate the icon-only / avatar / FAB changes; keep macOS labeled and FAB-less.
- For the Profile-tab avatar, the existing `SessionManager` (or the active `AccountStore`) already has the viewer's `ProfileBasic` with `avatar` URL — read from there. Cache via `AsyncImage`'s default LRU is fine.
- The FAB should use `.zIndex` and `.safeAreaInset` (or a `ZStack` overlay on the tab content) so it floats above the feed without breaking pull-to-refresh or scroll-to-top behavior on the underlying view.
- Verify scroll-to-top still works on the Home tab when the user taps the Home icon while already on Home — RN behavior — once the icon-fill swap is in.

## Acceptance

- iOS: tab bar shows five icon-only tabs; selected tab uses a filled icon; Profile tab shows the user's avatar; floating compose FAB appears at bottom-right on Home/Search/Notifications/Profile and opens `ComposerSheet`.
- macOS: unchanged from current behavior — labels remain, no FAB, Profile tab uses the existing icon.
- `swift build` clean; `xcodebuild` for both iOS Simulator and macOS clean.
