# 0002 — Home feed posts not loaded after sign-in

| | |
|---|---|
| **Status** | resolved |
| **Module** | BlueskyFeed |
| **Platform** | All |
| **First seen** | 2026-04-29 |
| **Closed** | 2026-04-29 |

## Description

After signing in, the home feed screen shows no posts. Two distinct fetch errors were logged: the primary timeline request was cancelled before it completed, and a secondary decoding failure occurred for post embeds.

## Steps to reproduce

1. Launch the app.
2. Sign in with a valid Bluesky account.
3. Observe the home feed.

## Expected behavior

The home feed loads and displays the user's timeline posts.

## Actual behavior

No posts were displayed. Two fetch errors appeared in the console.

## Root cause

Three separate bugs conspired to produce a blank feed:

**1. `@State` mutation in `body` (`FeedView`)**
`FeedView.currentViewModel` was a computed property that wrote to `@State var viewModels` during `body` execution. SwiftUI treats `body` as a pure function; the mutation triggered "Modifying state during view update" undefined behaviour, tearing down and restarting the view and cancelling any in-flight network task (URLError -999).

**2. Missing `decodeIfPresent` fallback for `labels` / `quoteCount`**
`PostView.labels: [Label]` and `ProfileBasic.labels: [Label]` were non-optional, but the Bluesky API sometimes omits those keys. A missing key caused `JSONDecoder` to throw `keyNotFound`, propagating through `FeedViewModel` as a decoding error.

**3. `EmbedRecordRef` decode failure for non-post embedded records**
Some stored post records (feed generators, lists, tombstoned quotes) embed an `app.bsky.embed.record` whose `record` object is missing `uri` and/or `cid`. The decoder tried `c.decode(EmbedRecordRef.self, forKey: .record)` unconditionally, throwing `keyNotFound` for the entire timeline response.

## Fix

**`Sources/BlueskyFeed/FeedView.swift`** — Deleted the `currentViewModel` computed property; moved VM creation into the `.task(id:)` closure where `@State` mutation is safe.

**`Sources/BlueskyCore/Post.swift`** — Added custom `init(from:)` to `PostView` using `decodeIfPresent` for `labels` (default `[]`) and `quoteCount` (default `0`).

**`Sources/BlueskyCore/Profile.swift`** — Added custom `init(from:)` to `ProfileBasic`, `ProfileView`, and `ProfileDetailed` using `decodeIfPresent` for `labels`.

**`Sources/BlueskyCore/Embed.swift`** — Wrapped `EmbedRecordRef` decoding in `try?` in both `app.bsky.embed.record` and `app.bsky.embed.recordWithMedia` cases; falls through to `.unknown(type)` when the record object is missing required fields.

**`Sources/BlueskyFeed/FeedStore.swift`** — Replaced `print()` debugging with `logger.debug()` / `logger.error()` via OSLog.
