# 0053 — Like state does not persist after tapping the like button

| | |
|---|---|
| **Status** | in-progress |
| **Module** | BlueskyFeed |
| **Platform** | All |
| **First seen** | 2026-04-30 |

## Description

Tapping the like button on a post appears to fire a network request (multiple `_loadCurrentDID` calls visible in the log) but the like state does not stick — the heart returns to unfilled and the like count does not increment. This is a regression; liking was previously confirmed working.

## Steps to reproduce

1. Open the Home feed.
2. Tap the like (heart) button on any post.
3. Observe the heart briefly fills, then reverts to unfilled.
4. Check the Bluesky website — the post shows no new like.

## Expected behavior

The heart fills and stays filled. The like count increments. The like is visible on bsky.app.

## Actual behavior

The like state reverts. The only log output is repeated `_loadCurrentDID` calls — no success or error message from the like/unlike API call is logged.

## Notes

The log shows `_loadCurrentDID` called 4 times in rapid succession when the like button is tapped, suggesting multiple concurrent requests may be racing or the optimistic update is being overwritten by a background refresh. Possible causes:

1. The optimistic post update in `FeedViewModel.like()` is being wiped out by a concurrent `loadMore()` or `refresh()` call that replaces the `posts` array.
2. The `createRecord` XRPC call for `app.bsky.feed.like` is silently failing and the error is not being surfaced.
3. Issue #0049 (duplicate FeedViewModel) may be causing the second VM's state to overwrite the first's optimistic update.

Check `FeedStore.like()` and `FeedViewModel.like()` for error handling and confirm the optimistic update is not being clobbered.
