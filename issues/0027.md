# 0027 — Module 9 feature: hashtag and topic views not implemented

| | |
|---|---|
| **Status** | resolved |
| **Module** | BlueskySearch |
| **Platform** | All |
| **First seen** | 2026-04-29 |

## Description

Tapping a hashtag in a post or a trending topic should open a dedicated view showing posts for that tag. No such `HashtagView` or `TopicView` exists.

## Steps to reproduce

1. View any post containing a `#hashtag` link.
2. Tap the hashtag.

## Expected behavior

A `HashtagView` opens showing a feed of posts tagged with that hashtag, loaded via `app.bsky.feed.searchPosts` with the `q` parameter set to the tag.

## Actual behavior

Nothing happens — hashtag taps are not handled, or they open a Safari URL instead of an in-app view.

## Notes

`RichTextView` renders hashtag facets as tappable links. The tap action needs to be routed through a navigation destination to a new `HashtagView`. `SearchViewModel.searchPosts` can back it.
