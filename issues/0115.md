# 0115 — Notification settings missing per-type sub-screens, push permission UI, and email toggles

| | |
|---|---|
| **Status** | open |

| **Platform** | iOS / All |

## Description

`NotificationSettingsScreen` is a single screen with 6 toggles (likes, reposts, follows, mentions, replies, quotes). The React Native equivalent has:

- **Push permission gate** at the top: detects iOS notification permissions; if not granted, shows a "Push notifications are off — turn them on in Settings" admonition + a deep link to system Settings.
- **Per-type sub-screens** for each notification kind (Likes, Reposts, Mentions, Replies, Quotes, Follows, Subscribed Posts, Verifications). Each sub-screen has separate toggles for **push** and **email** delivery and an "Include from" filter (Anyone / People I follow / Verified accounts).
- **Email notifications** as a separate top-level group with their own toggles (digest, security alerts).
- **DM notifications** toggle.

## Steps to reproduce

1. Settings → Notifications.
2. Compare to RN: SwiftUI is one flat screen, RN is a hub with 8+ sub-screens.

## Expected behavior

Refactor `NotificationSettingsScreen` into a hub with NavigationLinks to per-type sub-screens. Each sub-screen has push toggle, email toggle, and "Include from" picker. Add a top-level push-permission admonition that detects iOS authorization status and links to system Settings when denied.

## Actual behavior

One flat screen, six toggles, no permission detection, no email options.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Settings/NotificationSettings/` and the per-type screens.
- iOS push permission: `UNUserNotificationCenter.current().getNotificationSettings`.
- Files to edit/create: `BlueskyKit/Sources/BlueskySettings/NotificationSettingsScreen.swift` (turn into hub) + per-type sub-screens.
- Endpoints: `app.bsky.notification.putPreferences` (per-type filters live here under `app.bsky.notification.defs#preferences`).
