## Description

`StarterPackScreen` shows the header, member count, and a list of sample members. The React Native reference also displays:

- **Included feeds** — a section listing feed generators bundled in the pack with Pin / Add buttons.
- **Share** — a button that opens a system share sheet with the deep link (`https://bsky.app/starter-pack/<did>/<rkey>`).
- **QR code** — RN can render a QR for the deep link inside a sheet, useful for in-person sharing.
- **Copy link** — quick affordance for clipboard.

The "Follow All" button exists in SwiftUI, but the discoverability and shareability features are missing.

## Steps to reproduce

1. Open any starter pack.
2. Compare to RN.

## Expected behavior

Add to `StarterPackScreen`:

- **Included feeds** section between the header and member list, each row a `FeedCard` with a Pin button.
- **Share** button in the toolbar that presents a `ShareLink` with the deep-link URL.
- **QR code** option inside the share sheet (an image generated from the deep-link URL).
- **Copy link** in the share sheet.

## Actual behavior

Header + member list + Follow All only.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/StarterPack/StarterPackScreen.tsx`.
- QR generation: `CIQRCodeGenerator` filter on iOS, or one of the small Swift QR helpers.
- File to edit: `BlueskyKit/Sources/BlueskyLists/StarterPackScreen.swift`.

## Root cause

`StarterPackScreen` only rendered the header, member sample, and a Follow All button. RN's `StarterPackScreenInner` additionally renders a Feeds tab when `starterPack.feeds` is non-empty, and exposes Share / Copy link / QR code via `ShareDialog` + `QrCodeDialog`. The SwiftUI port had not been wired up for the share affordances even though `StarterPackView.feeds` was already decoded as `[GeneratorView]`.

## Fix

Extended `StarterPackScreen` with:

- A new `Feeds` section (between header and members) that renders each `GeneratorView` from `pack.feeds` using the existing `FeedCard` from `BlueskyUI`. The section is only shown when feeds are present, matching RN's `showFeedsTab = Boolean(starterPack.feeds?.length)` gate.
- A toolbar `Menu` (primaryAction) with three rows:
  - `ShareLink` for the canonical deep link.
  - `Copy link` button using `UIPasteboard` / `NSPasteboard`, with a transient toast confirmation.
  - `Show QR code` action presenting a sheet.
- `StarterPackQRSheet` renders the deep link as a QR via `CIFilter.qrCodeGenerator()` (CoreImage), scaled 12x and bridged to SwiftUI `Image` via `UIImage` / `NSImage` per platform. The sheet also exposes a `ShareLink` for parity with RN's QR dialog footer.
- Deep-link helper `shareURL(for:)` builds `https://bsky.app/start/<creator-handle>/<rkey>` using `ATURI.rkey`. RN reference: `src/lib/routes/links.ts#makeStarterPackLink`. Falls back to the creator's DID when the handle is empty.

## Files changed

- `BlueskyKit/Sources/BlueskyLists/StarterPackScreen.swift` — added Feeds section, toolbar share Menu, copy-to-clipboard with toast, and `StarterPackQRSheet`.

## Gotchas

- `CIFilter.qrCodeGenerator()` requires `import CoreImage.CIFilterBuiltins` — without it `filter.message` / `filter.correctionLevel` don't exist.
- The QR `outputImage` is tiny (~25×25 px); without `transformed(by:)` scaling and `interpolation(.none)` on the SwiftUI `Image` it renders blurry. Scale up first, then disable interpolation.
- Toast modifier in `BlueskyUI` is `toast(isPresented:message:)`, not a single optional message — using a `@State Bool` is required.
- RN `makeStarterPackLink` uses the creator handle, not the DID, even though the route accepts both. Match handle-first, fall back to DID.
