# 0096 — Composer character counter is plain text instead of a circular progress ring

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyComposer |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native composer renders the character counter as a circular progress ring (RN uses `ProgressCircle` / `ProgressPie`) that fills as the post grows toward the 300-character cap. The ring color shifts from primary to amber when fewer than 20 characters remain and to red when the post is over the limit. The numeric count appears inside or beside the ring.

The SwiftUI composer (`ComposerSheet.swift` lines 569–574) shows only a plain `Text` count. There's no ring, no color transition, and the affordance reads as a generic counter rather than the at-a-glance gauge RN provides.

## Steps to reproduce

1. Open the composer and start typing.
2. Compare the character indicator to the RN reference.

## Expected behavior

A circular progress ring sized roughly 24–28pt, drawn next to the Post button. As `text.count` grows toward 300:

- 0% — empty ring outline.
- 0–93%: ring fills primary blue.
- 93–100% (last ~20 chars): ring fills amber + numeric remaining count appears inside the ring.
- ≥100%: ring fills red, count goes negative, Post button disables.

Match RN's color thresholds and stroke weight.

## Actual behavior

Plain text count. No visual gauge, no color escalation.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/com/composer/text-input/CharProgress.tsx`.
- File to edit: `BlueskyKit/Sources/BlueskyComposer/ComposerSheet.swift`.
- Implementation note: SwiftUI `Circle().trim(from:to:).stroke(...)` rotated -90° is the natural way to draw the ring. Use `theme.colors.accent`, `theme.colors.warning`, `theme.colors.error` for the three states.
