# 0070 — iOS Profile screen diverges significantly from the React Native reference

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyProfile / Bluesky-SwiftUI |
| **Platform** | iOS |
| **First seen** | 2026-05-05 |

## Description

Live testing of the SwiftUI iOS Profile tab against the official Bluesky React Native app shows broad divergence. The most visible problems: a giant handle title floats above the banner, the stats row puts "Following" first (RN puts "Followers" first), the "Followed by …" known-followers chip is shown on the user's own profile (it should only appear on other users' profiles), pinned posts are not rendered, the Videos tab is missing from the segmented selector, and the profile-feed surface inherits all the same iOS chrome gaps tracked in #0068 (labeled bottom tab bar, generic profile-tab icon, no compose FAB).

## Side-by-side gap audit

### Banner area

| Area | SwiftUI (current) | RN (reference) |
|---|---|---|
| Handle as title | "brennan.sstools.co" rendered as a giant white headline ABOVE the banner image, taking ~15% of the screen | No headline title — the banner runs edge-to-edge under the status bar |
| Banner height | Pushed below the giant title; banner image sits in the middle third of the visible header | Banner sits flush under the status bar; tall, edge-to-edge |
| Avatar | Overlaps banner bottom-left, with a thick light ring | Overlaps banner bottom-left, no ring |
| Top-right action | Single ellipsis button on a circle | "Edit Profile" pill **plus** ellipsis button with a blue dot indicator (signaling something new in the menu) |

### Profile info row

| Area | SwiftUI | RN |
|---|---|---|
| Order of stats | `359 Following · 286 Followers · 1,424 Posts` — Following first, full-precision counts | `286 followers · 359 following · 1.4K posts` — Followers first, abbreviated counts |
| Number formatting | Group separator (`1,424`) | Abbreviated (`1.4K`) |
| Stats label case | Title case ("Following", "Followers", "Posts") | Lower case ("followers", "following", "posts") |
| Edit Profile button | Above the bio, sitting next to the avatar at the right edge (just under the banner) | Above the name, in its own right-aligned row, accompanied by the ellipsis-with-blue-dot |
| Bio block | Same content | Same content |
| URL formatting | "https://brennan.sstools.co" (with protocol) | "brennan.sstools.co" (protocol stripped for display) |
| "Followed by …" chip | **Shown on own profile** ("Followed by @ibrews.bsky.social, @martinp7r.com, and others you follow") — a bug | Not shown on own profile (only appears on other users' profiles) |

### Tab strip

- **SwiftUI**: `Posts · Replies · Media · Likes · Feeds · Lists` — six tabs visible, **no Videos tab**.
- **RN**: `Posts · Replies · Media · Videos · Likes · Feeds` — six tabs visible, plus Lists likely scrolling off the right edge. Videos tab is present.
- **Order**: After Media, SwiftUI jumps straight to Likes; RN inserts Videos between Media and Likes.
- **Pill background**: SwiftUI uses a pill-shaped capsule background around the entire tab strip; RN renders the tabs inline with just an underline on the selected tab.

### Pinned post

- **SwiftUI**: Pinned posts are not rendered. The first row in the Posts tab is "TIL: WebGPU…" from 3 hours ago — no pin indicator.
- **RN**: A `📌 Pinned` row precedes the first chronological post. The pinned post (Jan 3, 2025: "San Francisco is a magical place. #SF #PotreroHill" with photo) sits at the top with the pin icon in the byline.

### Bottom tab bar and compose entry

These match #0068 — labeled tabs vs icon-only, generic profile-tab icon vs user avatar, no compose FAB vs floating compose FAB. Track under #0068, not here.

## Notes

- "Followed by …" on own profile is a clear bug — the chip is meant to surface social context for *other* people's profiles ("here are mutuals you both follow"). Showing it on the viewer's own profile is meaningless.
- Reference: `Bluesky-ReactNative/src/view/screens/Profile.tsx`, `src/view/com/profile/ProfileHeader.tsx`, and `src/view/com/profile/ProfileSubpage.tsx`.
- Suggested priority: (4) the bug first, then (1)+(2) since they're the most jarring visually, then (7) pinned posts, then the rest.
