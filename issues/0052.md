# 0052 — Deep link opens a new window instead of routing in the existing window

| | |
|---|---|
| **Status** | in-progress |
| **Module** | Bluesky-SwiftUI |
| **Platform** | macOS |
| **First seen** | 2026-04-29 |

## Description

Opening a `bluesky://` URL (e.g. `open "bluesky://profile/brennan.sstools.co"`) causes macOS to open a new app window rather than routing within the existing window. The `onOpenURL` handler in `MainTabView` is called but the SwiftUI scene machinery creates a second window instead of reusing the first.

## Steps to reproduce

1. Build and run the app on macOS.
2. In Terminal, run: `open "bluesky://profile/brennan.sstools.co"`
3. Observe that a second app window appears instead of the existing window navigating to the profile.

## Expected behavior

The existing window handles the URL and navigates to the appropriate screen (profile, thread, etc.) without opening a new window.

## Actual behavior

A new app window is created.

## Notes

This is a macOS SwiftUI scene management issue. By default, `DocumentGroup` or multiple `WindowGroup` scenes will open a new window per URL. Fixes to investigate:

1. In `Bluesky_SwiftUIApp.swift`, ensure only a single `WindowGroup` is declared and it is not configured to open new windows for URLs.
2. Add `.handlesExternalEvents(matching: Set(arrayLiteral: "*"))` to the `WindowGroup` so the OS routes URLs to the existing window rather than spawning a new one.
3. On the `MainTabView` or its scene, add `.handlesExternalEvents(preferring: Set(arrayLiteral: "*"), allowing: Set(arrayLiteral: "*"))` to opt the existing window in as the URL handler.

Universal links would exhibit the same behavior since the underlying scene routing is the same.
