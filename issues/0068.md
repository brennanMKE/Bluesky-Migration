## Description

Live testing of the SwiftUI iOS build against the official Bluesky React Native app reveals broad divergence on the Home feed surface — top chrome, tab strip, post cards, action bar, and bottom tab bar all differ. Many of these gaps are platform-conventions that were never wired up because development was macOS-led, and individual pieces will be tracked as their own follow-up issues. This issue captures the side-by-side audit so the gaps can be triaged together.

## Side-by-side gap audit

### Top chrome

| Area | SwiftUI (current) | RN (reference) |
|---|---|---|
| Title | Large "Home" headline below the chrome | No screen title — replaced by tab strip |
| Logo | None | Bluesky butterfly logo, centered |
| Left action | None | Hamburger / drawer-menu button |
| Right action | Compose pencil + list/filter icon clustered in a pill at the very top | `#` hashtag (search) icon |
| Filter icon | Right edge of tab strip | Not present in this position |

### Tab strip (feeds switcher)

- **SwiftUI**: Two tabs only — "Following" and "Discover". A filter icon sits to the right of the tab strip.
- **RN**: Four+ tabs in a horizontal scroller — "Following · Mentions · Discover · Popular With Fri…" (truncates off-screen, scrollable). No filter icon adjacent to the tabs.

### Inline composer prompt

- **SwiftUI**: Absent.
- **RN**: A "What's up?" row sits at the very top of the feed under the tab strip — left avatar, prompt text, then camera and photo icons on the right. Tapping anywhere opens the full composer.

### Post card layout

| Area | SwiftUI | RN |
|---|---|---|
| Author row separators | "NBA @nba.com   2m" with no separator | "NBA ✓ @nba.com · 7m" with bullet separator and verified badge |
| Verified badge | Not shown on the NBA account | Blue checkmark next to display name |
| Image embed | Inset with visible side margins; rounded corners | Near-edge-to-edge with smaller side margin; rounded corners |
| Truncated handle | "@yodelinghippie.bsk…" | "@yodelin…" (more aggressive truncation) |

### Post action bar

- **Order is different.** SwiftUI: `reply · repost · like · share · bookmark · ellipsis`. RN: `reply · repost · like · bookmark · share · ellipsis` — bookmark before share.
- **Repost count missing in SwiftUI.** RN shows the count next to the repost icon (e.g. `2`); SwiftUI shows the icon only.
- **Like count present in both** — but RN shows the heart-tap-count even when zero; SwiftUI seems to omit zero.
- **Share icon shape differs.** SwiftUI uses an upload-arrow box; RN uses a forward-arrow.

### Bottom tab bar

| Area | SwiftUI | RN |
|---|---|---|
| Tab icons | Home · Search · Messages · Notifications · Profile (icon + text label, all five labeled) | Home · Search · Messages · Notifications · Profile (icon-only — no text labels) |
| Profile tab icon | Generic person silhouette | The user's own avatar |
| Selected indicator | Pill background behind selected icon + label | Just the icon turns solid (filled vs outline) |
| Compose entry | Top-right pencil icon | Floating action button (FAB) bottom-right, blue circle with pencil — overlaps the feed |

## Notes

- This is iOS-only — the equivalent macOS surface has different platform conventions and shouldn't change to match the iOS RN screenshots.
- Most of these gaps are shipped behavior in the RN client and likely originate from `Bluesky-ReactNative/src/view/screens/Home.tsx` plus `src/view/com/posts/`. Use that tree as the source of truth for any disputed behavior.
- This audit is intentionally broad. Don't try to fix everything at once — file the items above as discrete issues and pick the highest-impact ones first. Recommended order: (8) bottom tab bar, (5) action bar reorder, (1) top chrome, (3) feed tabs, (2) inline composer, then the rest.
