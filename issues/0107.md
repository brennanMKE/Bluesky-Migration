# 0107 — Message thread missing per-message timestamps and date dividers

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyMessages |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

Two related gaps in the SwiftUI message thread:

1. **No per-message timestamps.** RN shows a relative timestamp under each bubble (or under the last bubble in a clustered run from one sender). SwiftUI bubbles have no timestamp at all.
2. **No date dividers.** RN renders a centered date pill ("Today", "Yesterday", "Mar 4") between groups of messages from different days. SwiftUI's `LazyVStack` is one continuous list with no day separators.

Together these make it hard to anchor a conversation in time.

## Steps to reproduce

1. Open a message thread that spans multiple days.
2. Scroll through.
3. Observe: no timestamps under bubbles, no date dividers between days.

## Expected behavior

- **Per-message timestamp**: under each bubble (or the last bubble in a run from the same sender, RN's clustering rule). Format: relative for today ("2h ago"), time-only for older same-day, day+time for older days. Match RN's `niceDate` helper for consistency.
- **Date dividers**: a small centered pill ("Today" / "Yesterday" / "Mar 4") above the first message of a new day. The pill uses the secondary text color and a subtle background.

## Actual behavior

No timestamps, no dividers.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Messages/components/DateDivider.tsx` and the timestamp rendering in `MessageItem.tsx`.
- File to edit: `BlueskyKit/Sources/BlueskyMessages/MessageThreadScreen.swift`. The `LazyVStack` ForEach should be re-grouped so dividers can be inserted between adjacent messages whose `sentAt` falls on different calendar days.
- Helper: a small `Calendar.current.isDate(_:inSameDayAs:)` check works.
