## Description

Manual sign-off for the Settings tree — confirm every RN sub-screen
exists, persists across launches, and the destructive flows (sign out,
deactivate account, delete account) work.

## Test sequence

1. **Open Settings.** From Profile → `…` → Settings. Confirm the
   landing screen lists every section RN ships: Account · Privacy
   & Security · Notifications · Content & Media · Accessibility ·
   Languages · Appearance · Help · About.
2. **Account.**
   - Email change, password change, handle change, birthday, deactivate,
     export data (#0113) — each is a sub-screen that opens correctly
     and saves a change.
   - Sign-out button at the bottom, destructive-styled, returns to
     sign-in screen.
3. **Privacy & Security.**
   - 2FA toggle is inline (no separate sub-screen for the toggle).
   - App passwords link with a count badge.
   - "Logged-out view (PWI) opt-out" toggle (#0114).
4. **Notifications.**
   - Per-type sub-screens (likes / reposts / replies / mentions /
     follows / quotes) (#0115).
   - Push permission CTA on iOS if not granted.
   - Email notification toggles.
5. **Thread Preferences.** Sort order radio (Hot / Newest / Oldest /
   Random — RN parity), Group replies toggle (#0116).
6. **Following Feed Preferences.** Hide replies / Hide reposts /
   Hide quotes (#0117).
7. **External Media Preferences.** Per-source toggles for YouTube /
   Spotify / Twitch / Tenor (#0118).
8. **Languages.** App language and content language multi-select
   (#0120).
9. **Accessibility.** Larger alt-text badge toggle, haptic feedback
   toggle (iOS only) (#0121).
10. **Activity Privacy.** Who can DM / mention / interact (#0122).
11. **Content Filtering / Moderation.** Adult content toggle,
    labelers, content-warning preferences.
12. **About.**
    - System info section (version, build, OS).
    - Status link to status.bsky.app.
    - Dev-mode tools (debug logs, copy DID, etc.) (#0124).
13. **Persistence test.** Change two settings in different sections.
    Quit and relaunch the app. Confirm both changes survive
    (#0037, #0065).
14. **Sign out from Settings.** Tap Sign Out. Confirm the sign-in
    screen appears and the keychain entry for that account is
    cleared.

## Sign-off checklist

- [ ] Landing screen lists every RN section
- [ ] Account sub-screen — every action works
- [ ] Privacy & Security toggles and links work
- [ ] Notifications sub-screens + push permission work
- [ ] Thread Preferences saves
- [ ] Following Feed Preferences saves
- [ ] External Media Preferences saves
- [ ] Languages multi-select saves
- [ ] Accessibility toggles save
- [ ] Activity Privacy saves
- [ ] About has version + status + dev tools
- [ ] All changes persist across relaunch
- [ ] Sign out works
- [ ] No regressions in #0037, #0065, #0113–#0124
