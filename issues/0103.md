# 0103 — Composer video attachment is missing the captions / subtitles upload

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyComposer |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native composer lets a video attachment carry both alt text *and* a captions track (WebVTT). RN's `SubtitleDialogBtn` opens a `SubtitleDialog` where the user can upload a `.vtt` file (or paste captions inline) and the captions blob is uploaded alongside the video and referenced from `app.bsky.embed.video.captions`.

The SwiftUI composer supports alt text on videos but has no captions UI. A SwiftUI-posted video cannot carry captions, which fails accessibility parity with RN.

## Steps to reproduce

1. Open the SwiftUI composer, attach a video.
2. Look for a "Captions" / "Subtitles" affordance next to the alt-text editor.
3. Observe there is none.

## Expected behavior

Per video, in addition to the alt-text editor:

- A "Captions" button that opens a sheet.
- The sheet supports: pick a `.vtt` file from Files, or paste inline VTT text into a `TextEditor`.
- Captions count appears next to the button (e.g. "Captions (en)" once attached).
- On post submit, the captions text is uploaded as a blob and referenced in `app.bsky.embed.video.captions[]` with the language code (default to the post's primary language).

## Actual behavior

No captions UI. Videos post with alt text only.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/com/composer/videos/SubtitleDialog.tsx`.
- Lexicon: `app.bsky.embed.video#caption` (`lang: string`, `file: blob`). Confirm `BlueskyCore`'s `EmbedVideo` has the `captions` field; if not, add it.
- File to edit: `BlueskyKit/Sources/BlueskyComposer/ComposerSheet.swift` (the video preview block around lines 340–369), and `ComposerViewModel.swift` (extend `uploadVideoBlob` to also upload captions blobs).
