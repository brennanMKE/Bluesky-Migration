# 0112 — Compose bar in DMs doesn't detect mentions, links, or hashtags (no facets)

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyMessages |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

When a user types `@alice` or `https://example.com` or `#tech` in the React Native message composer, RN runs the same facet-builder used by the post composer (`useExtractEmbedFromFacets`) and:

- Resolves `@handle` to a DID and includes a mention facet in the sent message.
- Detects URLs and either includes a link facet, or upgrades the message to include a link-card embed.
- Highlights mentions/links inline as the user types.

The SwiftUI compose bar (`MessageThreadScreen.swift` ~lines 104–128) sends the raw text via `MessageInput.text` without facets and without inline highlighting. Recipients see plain text where mentions and links should be tappable.

## Steps to reproduce

1. Open the SwiftUI compose bar.
2. Type `Hey @alice.bsky.social check out https://example.com`.
3. Send.
4. View the sent message in the recipient's RN client.
5. Observe @alice is plain text, not a mention; the URL is plain text, not a link.

## Expected behavior

The compose bar runs `FacetBuilder` (already exists in `BlueskyComposer`) over the input text on send:

- `@handle` segments are resolved to DID and added as `mention` facets.
- URL segments are added as `link` facets.
- `#tag` segments are added as `tag` facets.
- The outgoing `MessageInput` carries `text` + `facets[]`.

For visual polish (lower priority but RN does it): live-highlight detected facets with the link color while the user is typing. This can use the same `RichTextView`-based path as the post composer, or a simpler inline regex check.

## Actual behavior

Raw plaintext is sent. Facets are never built. Inline highlighting is absent.
