# 0099 — Composer is missing the GIF picker (Tenor)

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyComposer |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native composer toolbar includes a GIF button (`SelectGifBtn`) that opens a Tenor-backed GIF search dialog. The selected GIF is attached to the post as an external embed (`app.bsky.embed.external` pointing at the Tenor URL with thumbnail). The SwiftUI composer toolbar has only image and video pickers — no GIF support.

Tracked separately from #0034 (which lists video picker / link card preview / threading / drafts) because GIF is a distinct media path with its own search UI and embed type.

## Steps to reproduce

1. Open the SwiftUI composer.
2. Look for a GIF button in the media toolbar.
3. Observe there is none.

## Expected behavior

A GIF button next to the image / video buttons in the composer media toolbar. Tapping opens a sheet:

- Search bar with debounced Tenor query.
- Trending GIFs section when query is empty.
- Grid of animated GIF previews; tap to select.
- Selected GIF appears in the composer's embed area with a "Remove" affordance.

On post creation, the GIF is attached as an external embed (`app.bsky.embed.external` with `uri`, `title`, `description`, and `thumb` blob).

## Actual behavior

No GIF button. Only image and video pickers exist.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/com/composer/gifs/SelectGifBtn.tsx` and the GIF search dialog implementation.
- Tenor API: requires a Tenor API key. RN reads it from the build env. SwiftUI side would need an equivalent config slot in `Bluesky-SwiftUI/Config.swift` or `Build.xcconfig`.
- Embed type: `app.bsky.embed.external` (already supported in `BlueskyCore` for link cards). Reuse — no new lexicons needed.
- File to edit: `BlueskyKit/Sources/BlueskyComposer/ComposerSheet.swift`. Add a `GIFPickerSheet.swift` view + a small Tenor client.
