# 0055 — Switching tabs while in a thread does not change the active tab

| | |
|---|---|
| **Status** | in-progress |
| **Module** | Bluesky-SwiftUI |
| **Platform** | macOS |
| **First seen** | 2026-04-30 |

## Description

When the Home tab is showing a thread view (pushed via NavigationStack), clicking a different tab in the sidebar (e.g. Profile) does not switch to that tab. The thread view remains visible. The sidebar highlights the new tab but the detail pane does not change.

## Steps to reproduce

1. Open the app and tap any post to open the thread view.
2. Click "Profile" in the sidebar.
3. Observe that the thread view is still displayed — Profile does not appear.
4. Press the back button to return to the feed — only then does the Profile screen appear.

## Expected behavior

Each tab has its own independent navigation stack. Clicking a sidebar tab immediately switches the detail pane to that tab's root screen, regardless of what is pushed on any other tab's stack.

## Actual behavior

The detail pane stays on the thread view until the user manually pops back to the feed. The tabs share a single NavigationStack instead of having independent stacks per tab.

## Notes

On macOS, `MainTabView` uses a `NavigationSplitView` with a single `NavigationStack` wrapping `tabContent(selectedTab)`. When `selectedTab` changes, the stack's root changes but the existing pushed views may not be cleared. The fix is to ensure each tab's navigation stack is isolated — either by keying the `NavigationStack` on the selected tab (`.id(selectedTab)`) so it resets when the tab changes, or by maintaining a separate `NavigationPath` per tab.
