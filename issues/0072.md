## Description

The iOS Home tab currently shows a giant "Home" headline taking ~25% of the screen above the feed. The React Native reference replaces this with a slim, fixed top bar containing three controls — a hamburger button on the left (opens a side drawer), the Bluesky butterfly logo centered, and a `#` button on the right (navigates to a "My Feeds" screen). Mirroring this layout is the simplest, highest-impact change to make the iOS UI feel like the official Bluesky app.

This is part of the larger iOS parity audit in #0068.

## Scope

### Top bar layout

- **Drop the large "Home" navigation title** on iOS. macOS layout is unchanged.
- **Add a slim top bar** that sits flush below the status bar and contains:
  - **Left**: hamburger / drawer button (`line.3.horizontal` SF Symbol). Opens the side drawer (see below).
  - **Center**: Bluesky butterfly logo at small size — the official Bluesky brand mark. Use the existing app icon or vector if available, otherwise add a properly licensed SVG/PNG asset to the asset catalog.
  - **Right**: `#` button (`number` SF Symbol or a custom hash icon) — navigates to the "My Feeds" screen, tracked in #0073.
- **The compose pencil + list/filter icon cluster** that currently lives top-right is removed on iOS. Compose moves to the FAB (#0071); the filter icon moves elsewhere or is dropped if redundant with the feed-tabs scroller (#0074).

### Side drawer

The hamburger button opens a drawer (slide-in from the left) with:
- The user's profile card at top (avatar, display name, handle).
- Navigation rows: Profile · Search · Feeds · Lists · Bookmarks · Settings · Sign out.
- Honor RN's drawer ordering and iconography (see `Bluesky-ReactNative/src/view/shell/Drawer.tsx`).

### Top bar visibility

- The bar stays pinned at all times on Home (does not collapse on scroll). RN behavior — verify this is the desired behavior on iOS too; if collapse-on-scroll is preferred, file a follow-up.
- On other screens (Search, Notifications, Profile, Messages), each screen's top bar pattern is its own concern — those are tracked under #0069 / #0070 / etc.

## Implementation notes

- Add a reusable `BlueskyTopBar` view in `BlueskyUI` so it can be shared across screens that need the same layout. Take left/center/right slots so each screen can supply its own actions.
- The drawer can be a `NavigationSplitView` sidebar on iPad/macOS and a custom slide-in `ZStack` overlay on iPhone. RN uses a custom drawer; iOS-native equivalent is a `presentationDetents`-based sheet from the leading edge or a hand-rolled overlay.
- Wire the `#` button to push the My Feeds screen (#0073). Until that lands, the button can route to a placeholder view; do not silently no-op.
- Asset for the butterfly logo: confirm the brand-asset license. The Bluesky butterfly is a registered trademark; an open-source third-party client should be careful about how it uses it. May need to use a generic icon or text mark instead — flag for user before shipping.

## Acceptance

- iOS Home shows a slim top bar with hamburger / butterfly / hash, no large "Home" headline.
- Hamburger opens a working drawer with the listed navigation rows.
- `#` button pushes My Feeds (or a placeholder) — see #0073.
- macOS layout is unchanged.
- `swift build` clean; iOS Simulator and macOS builds clean.

## Open questions

- **Butterfly logo licensing.** Confirm whether using the official Bluesky butterfly mark in a third-party SwiftUI client is acceptable. If not, fall back to a wordmark ("Bluesky") or a neutral logomark.
- **Drawer presentation style.** Hand-rolled slide-in vs `NavigationSplitView` sidebar — choose based on iPad considerations.
