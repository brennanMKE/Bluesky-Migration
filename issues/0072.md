## Description

The iOS Home tab currently shows a giant "Home" headline taking ~25% of the screen above the feed. The React Native reference replaces this with a slim, fixed top bar containing three controls — a hamburger button on the left (opens a side drawer), the Bluesky butterfly logo centered, and a `#` button on the right (navigates to a "My Feeds" screen). Mirroring this layout is the simplest, highest-impact change to make the iOS UI feel like the official Bluesky app.

This is part of the larger iOS parity audit in #0068.

## Scope

### Top bar layout

- **Drop the large "Home" navigation title** on iOS. macOS layout is unchanged.
- **Add a slim top bar** that sits flush below the status bar and contains:
  - **Left**: hamburger / drawer button (`line.3.horizontal` SF Symbol). Opens the side drawer (see below).
  - **Center**: Bluesky butterfly logo at small size — the official Bluesky brand mark. Use the existing app icon or vector if available, otherwise add a properly licensed SVG/PNG asset to the asset catalog.
  - **Right**: `#` button (`number` SF Symbol or a custom hash icon) — navigates to the "My Feeds" screen, tracked in #0073.
- **The compose pencil + list/filter icon cluster** that currently lives top-right is removed on iOS. Compose moves to the FAB (#0071); the filter icon moves elsewhere or is dropped if redundant with the feed-tabs scroller (#0074).

### Side drawer

The hamburger button opens a drawer (slide-in from the left) with:
- The user's profile card at top (avatar, display name, handle).
- Navigation rows: Profile · Search · Feeds · Lists · Bookmarks · Settings · Sign out.
- Honor RN's drawer ordering and iconography (see `Bluesky-ReactNative/src/view/shell/Drawer.tsx`).

### Top bar visibility

- The bar stays pinned at all times on Home (does not collapse on scroll). RN behavior — verify this is the desired behavior on iOS too; if collapse-on-scroll is preferred, file a follow-up.
- On other screens (Search, Notifications, Profile, Messages), each screen's top bar pattern is its own concern — those are tracked under #0069 / #0070 / etc.

## Implementation notes

- Add a reusable `BlueskyTopBar` view in `BlueskyUI` so it can be shared across screens that need the same layout. Take left/center/right slots so each screen can supply its own actions.
- The drawer can be a `NavigationSplitView` sidebar on iPad/macOS and a custom slide-in `ZStack` overlay on iPhone. RN uses a custom drawer; iOS-native equivalent is a `presentationDetents`-based sheet from the leading edge or a hand-rolled overlay.
- Wire the `#` button to push the My Feeds screen (#0073). Until that lands, the button can route to a placeholder view; do not silently no-op.
- Asset for the butterfly logo: confirm the brand-asset license. The Bluesky butterfly is a registered trademark; an open-source third-party client should be careful about how it uses it. May need to use a generic icon or text mark instead — flag for user before shipping.

## Root cause

The iOS `MainTabView` rendered the Home tab inside a stock SwiftUI
`NavigationStack` whose toolbar/title chrome was the system large-title
nav bar. `FeedView.navigationTitle("Home")` then surfaced the giant
"Home" headline, and the toolbar carried both a compose pencil and a
`list.star` saved-feeds shortcut on the trailing edge — none of which
match the React Native reference's slim hamburger / brand / `#` bar.
There was also no shared component for screens to opt into the slim
layout.

## Fix

Added a reusable `BlueskyTopBar` view in `BlueskyUI` (Layer 2) with
leading / centre / trailing slots and a default `BlueskyWordmark` in
the centre. The view is a fixed-height (44pt) `HStack` with a theme
background and a hairline bottom border — RN-style flush-below-status-bar
chrome that any screen can mount.

Wired the iOS compact `MainTabView` layout to:

- Suppress the system nav bar's large title on iOS (`navigationBarTitleDisplayMode(.inline)` from `FeedView`) so macOS keeps its `navigationTitle("Home")` but iOS gets no headline.
- Mount `BlueskyTopBar` above `tabContent(.home)` inside the
  `NavigationStack`, with a hamburger button (`line.3.horizontal`) that
  toggles a hand-rolled slide-in drawer overlay, the wordmark in the
  centre, and a trailing `#` button (`number`) that pushes a
  placeholder `MyFeedsPlaceholderView` while #0073 is open.
- Gate the existing compose-pencil and `list.star` toolbar items behind
  `horizontalSizeClass == .regular` on iOS so iPhone no longer shows
  them; iPad regular and macOS keep their existing toolbar shortcuts.

The drawer is a `ZStack` overlay with a tap-to-dismiss scrim and an
opaque panel (~82% screen width, capped at 340pt) sliding in from the
leading edge. It contains a profile card (avatar, display name, handle)
sourced from the same per-session `getProfile` fetch the FAB uses, plus
nav rows for Profile, Search, Feeds, Lists, Bookmarks, Settings, and a
destructive Sign out at the bottom. Rows that target a per-tab
destination (Lists, Bookmarks, Settings) switch to the relevant tab
first, then push the destination after a short delay so the drawer
slide-out animation doesn't fight the `navigationDestination` push.

## Gotchas

- **Brand mark choice — wordmark, not butterfly.** The asset catalog
  has no licensed Bluesky butterfly mark, and the issue flags the
  butterfly as a registered trademark with unclear third-party-client
  use. To avoid trademark risk and to ship something today rather than
  blocking on an art pass, the centre slot is a heavy "Bluesky"
  wordmark in the brand link colour — readable, distinct, and
  trivially swappable for an SF-Symbol or vector logo later if a
  licensed asset arrives. Confirm the wording / weight you want (or
  drop in a butterfly SVG and the centre slot just renders it
  instead).
- **Pinned, not collapse-on-scroll.** The bar is pinned at all times
  on Home, matching RN. Implementing collapse-on-scroll would require
  a `ScrollViewReader`-driven offset on `BlueskyTopBar` plus a way for
  child screens to publish their scroll offset, which is more
  invasive than this pass warrants. Filed implicitly under the parent
  audit #0068; reopen if you want it.
- **Filter dropdown left in place.** The inline "hide replies / hide
  reposts" filter menu in `FeedView` is still rendered above the post
  list; removing it would lose functionality, and the issue notes the
  filter belongs to the feed-tabs scroller (#0074) rather than the
  top bar. Track its eventual move there.
- **Drawer destinations use a 220ms delay before pushing.** Tapping a
  drawer row that targets a per-tab destination (e.g. Settings) first
  runs `dismissDrawer()`, then sleeps 220ms before flipping the
  matching `showSettings` flag. Without the delay, the drawer's
  slide-out animation and the `navigationDestination` push fight each
  other and the destination either doesn't appear or appears with a
  janky transition. The delay matches the drawer animation duration.
- **`UIScreen.main.bounds.width`** is used to size the drawer panel —
  fine in iOS-only code but would need a `GeometryReader` swap if the
  drawer ever becomes multi-platform.
