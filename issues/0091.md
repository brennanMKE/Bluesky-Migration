# 0091 — Signup flow is entirely missing from the SwiftUI app

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyAuth / Bluesky-SwiftUI |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native client supports new-account creation directly inside the app via a multi-step signup flow (`Bluesky-ReactNative/src/screens/Signup/`): **StepInfo** (email + password + birthdate), **StepHandle** (handle picker with availability check) and **StepCaptcha** (device attestation). The SwiftUI port has no signup UI at all — a brand-new user has nowhere to go from `LoginView`. The only way to create an account is to leave the app and use bsky.app, then return to log in.

For parity (and so the SwiftUI app is usable as a primary client), signup needs to be implemented as a SwiftUI flow that mirrors the RN steps and posts to the same `com.atproto.server.createAccount` endpoint.

## Steps to reproduce

1. Launch the SwiftUI app fresh (no saved account).
2. On `LoginView`, look for a way to create a new account.
3. Observe there is no "Create account" / "Sign up" affordance.

## Expected behavior

A "Create account" button on `LoginView` that opens a multi-step signup flow:

- **Step 1 — Info**: hosting provider picker (defaults to `bsky.social`), invite code (if required), email, password, birthdate.
- **Step 2 — Handle**: handle text field with live availability check, `.bsky.social` (or PDS) suffix preview, suggestions.
- **Step 3 — Captcha**: device attestation / hCaptcha equivalent. iOS already has access to `DCAppAttestService`, which is the natural fit; macOS may need a fallback.
- **Step 4 (deferred to onboarding, see #0092)**: profile setup → interests → suggested follows → completion.

Each step has back/next navigation, validates inputs locally, and displays server errors inline. On success the new session is saved via `SessionManager` exactly like login.

## Actual behavior

No signup affordance anywhere. `LoginView` is the only auth screen and only handles existing-account login.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Signup/index.tsx` (orchestrator) and `state/StepInfo.tsx`, `state/StepHandle.tsx`, `state/StepCaptcha.tsx`.
- AT Proto endpoint: `com.atproto.server.createAccount`. Lexicon types may need to be added to `BlueskyCore/Auth.swift` (CreateAccountRequest/Response, hosting-provider URL list, invite-code requirements).
- This issue is the first half of a two-part missing flow; #0092 covers the post-signup onboarding (interests, suggested follows, etc.) that RN runs immediately after a successful create-account.
- Files to create: new `SignupFlowView.swift` (container) + step views, in `BlueskyKit/Sources/BlueskyAuth/`. Wire into `LoginView` via a NavigationLink / sheet.
