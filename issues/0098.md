# 0098 — Composer is missing the threadgate / postgate (reply + quote permissions) picker

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyComposer |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native composer lets the author restrict who can reply to (`threadgate`) and who can quote (`postgate`) the post being created. RN exposes a `ThreadgateBtn` in the toolbar that opens a dialog with options: **Everyone**, **People you follow**, **People you mention**, **Lists** (multi-select of moderation lists), and **Nobody**. Quote permissions are similarly configurable.

These records are written alongside the post:

- `app.bsky.feed.threadgate` — controls reply permissions.
- `app.bsky.feed.postgate` — controls embed/quote permissions.

The SwiftUI composer has no threadgate / postgate UI. Every post is implicitly open to all replies and quotes, which is a privacy + control regression vs. RN.

## Steps to reproduce

1. Open the SwiftUI composer.
2. Look for a "Who can reply?" / "Who can quote?" affordance.
3. Observe there is none.

## Expected behavior

A reply-restrictions button in the composer toolbar (RN uses a chat-bubble icon) that opens a sheet:

- **Reply restrictions** (radio):
  - Everyone (default)
  - People you follow
  - People you mention
  - Combine: people in selected lists (with list multi-select)
  - Nobody
- **Quote restrictions** (toggle): Allow quotes (default on) / Disable quotes.

When the post is submitted, if restrictions are non-default, the app additionally writes:
- `app.bsky.feed.threadgate` record at the same `rkey` as the post (key insight: the threadgate `rkey` matches the post's `rkey`).
- `app.bsky.feed.postgate` record at the same `rkey` if quote restrictions differ from default.

## Actual behavior

No threadgate / postgate UI. Posts are created without companion records, so they are implicitly open.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/com/composer/threadgate/`.
- Lexicons to add to `BlueskyCore`: `ThreadgateRecord` (`allow: [Allow]`, where Allow is `mentioned` / `followers` / `list`), `PostgateRecord` (`embeddingRules`).
- File to edit: `BlueskyKit/Sources/BlueskyComposer/ComposerSheet.swift` and `ComposerViewModel.swift` — extend the create-post path to optionally write threadgate / postgate records via `applyWrites`.
- Reply restrictions also need to be *displayed* on the focal post in thread view ("Only people @op mentions can reply") — that's a separate render-side parity gap (see #0147 once filed) but mentioned here for context.
