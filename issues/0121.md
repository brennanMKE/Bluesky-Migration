## Description

The React Native Accessibility settings screen offers three controls SwiftUI doesn't:

- **Display larger alt-text badges** — toggle that increases the size of the small "ALT" badge on images, for low-vision users who want to know at a glance which images are described.
- **Disable haptic feedback** (iOS only) — toggle to disable taptic feedback on actions.
- **Require alt text** — already in SwiftUI but not consumed by composer (see #0101).

SwiftUI's `AccessibilitySettingsScreen` has Reduce Motion, Open Links In-App, and Require Alt Text, but is missing the larger-badge toggle and the haptics toggle.

## Steps to reproduce

1. Settings → Accessibility.
2. Compare to RN — SwiftUI has 3 controls, RN has 5+.

## Expected behavior

Add to `AccessibilitySettingsScreen`:

- **Display larger alt-text badges** — toggle. Consumed by `PostEmbedView` (image embed) which scales the ALT badge accordingly.
- **Disable haptic feedback** — iOS-only toggle. Consumed wherever the app calls `UIImpactFeedbackGenerator` / SwiftUI's `.sensoryFeedback(...)`.

## Actual behavior

Only Reduce Motion, Open Links In-App, Require Alt Text.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Settings/AccessibilitySettings/`.
- File to edit: `BlueskyKit/Sources/BlueskySettings/AccessibilitySettingsScreen.swift`.
- The haptics toggle should gate every call site; consider centralizing via a small `HapticsService` that reads the preference.

## Root cause

`AccessibilitySettingsScreen` only exposed three controls (Reduce Motion,
Open Links In-App, Require Alt Text). RN's `AccessibilitySettings.tsx` also
ships **Display larger alt text badges** and (iOS-only) **Disable haptic
feedback**, which had no SwiftUI-side equivalent — neither the toggle UI nor
the persisted preference key existed.

## Fix

- Added two persisted preferences to `SettingsViewModel`:
  - `largerAltBadge` (Bool, default `false`) under key `settings.largerAltBadge`.
  - `disableHaptics` (Bool, default `false`) under key `settings.disableHaptics`.
  Both load/save via the existing `loadValue` / `saveValue` helpers.
- `AccessibilitySettingsScreen` now renders:
  - An **Alt text** section with `Require alt text before posting` (relabeled
    to match RN copy exactly) plus the new `Display larger alt text badges`.
  - A `#if os(iOS)` **Haptics** section with the new `Disable haptic feedback`
    toggle. The macOS build omits the section entirely so the screen stays
    parity-correct with RN's `IS_NATIVE` gate.
- `.onChange` hooks on both new toggles fan out to `viewModel.save()` so
  values persist immediately, matching the existing toggle pattern.

## Files changed

- `BlueskyKit/Sources/BlueskySettings/SettingsViewModel.swift`
- `BlueskyKit/Sources/BlueskySettings/AccessibilitySettingsScreen.swift`

## Gotchas

- **Consumer wiring deferred for both toggles.** The preferences round-trip
  to `UserDefaultsPreferencesStore` but nothing currently reads them:
  - `BlueskyUI/PostEmbedView.swift` does not yet render an "ALT" badge on
    image embeds (it only sets `accessibilityLabel`). Once the badge is
    added, it should scale based on `largerAltBadge`.
  - The codebase has no `UIImpactFeedbackGenerator` / `.sensoryFeedback`
    call sites yet (grep returned no results in `BlueskyKit/Sources/` or
    `Bluesky-SwiftUI/`). When haptics are added, route them through a
    central `HapticsService` that consults `settings.disableHaptics` and
    no-ops when the preference is on.
- The `Require Alt Text` row was relabeled to `Require alt text before posting`
  to match RN's exact copy. The descriptive footer was retained for context.
