# 0121 — Accessibility settings missing larger-alt-badge toggle and haptic feedback toggle

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskySettings |
| **Platform** | iOS / All |
| **First seen** | 2026-05-06 |

## Description

The React Native Accessibility settings screen offers three controls SwiftUI doesn't:

- **Display larger alt-text badges** — toggle that increases the size of the small "ALT" badge on images, for low-vision users who want to know at a glance which images are described.
- **Disable haptic feedback** (iOS only) — toggle to disable taptic feedback on actions.
- **Require alt text** — already in SwiftUI but not consumed by composer (see #0101).

SwiftUI's `AccessibilitySettingsScreen` has Reduce Motion, Open Links In-App, and Require Alt Text, but is missing the larger-badge toggle and the haptics toggle.

## Steps to reproduce

1. Settings → Accessibility.
2. Compare to RN — SwiftUI has 3 controls, RN has 5+.

## Expected behavior

Add to `AccessibilitySettingsScreen`:

- **Display larger alt-text badges** — toggle. Consumed by `PostEmbedView` (image embed) which scales the ALT badge accordingly.
- **Disable haptic feedback** — iOS-only toggle. Consumed wherever the app calls `UIImpactFeedbackGenerator` / SwiftUI's `.sensoryFeedback(...)`.

## Actual behavior

Only Reduce Motion, Open Links In-App, Require Alt Text.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Settings/AccessibilitySettings/`.
- File to edit: `BlueskyKit/Sources/BlueskySettings/AccessibilitySettingsScreen.swift`.
- The haptics toggle should gate every call site; consider centralizing via a small `HapticsService` that reads the preference.
