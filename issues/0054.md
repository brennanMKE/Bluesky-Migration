# 0054 — Cannot tap a reply in thread view to navigate into it

| | |
|---|---|
| **Status** | in-progress |
| **Module** | BlueskyFeed |
| **Platform** | All |
| **First seen** | 2026-04-30 |

## Description

In the thread view, tapping on a reply post card does nothing. Each reply may itself have replies (sub-threads), but there is no way to drill into them because the tap gesture is not wired up.

## Steps to reproduce

1. Tap any post in the feed to open the thread view.
2. Scroll down to the replies listed below the focal post.
3. Tap on any reply post card.
4. Observe that nothing happens — no navigation occurs.

## Expected behavior

Tapping a reply navigates to that reply's own thread view, showing it as the focal post with its own replies below it.

## Actual behavior

The tap is silently ignored. Sub-thread replies are inaccessible.

## Notes

`ThreadView` renders replies as `PostCard` items with a `PostCard.Actions` that only wires `onReply`. The `onTap` closure is not set, so tapping the card body does nothing. The fix is to wire `onTap` in `ThreadView.actions(for:)` to push a new `ThreadView` via a `navigationDestination`, passing the tapped reply's URI. This requires a `@State var selectedReplyURI: ATURI?` and a `.navigationDestination(item:)` on the list.
