## Description

The hamburger-triggered sidebar drawer was added in #0072 with a small set of navigation rows. The React Native reference (attached) shows a much richer drawer with three distinct zones, and the SwiftUI version is missing most of them.

**RN drawer (top to bottom):**

1. **Profile header**
   - Avatar (circular, ~64pt).
   - Display name (large, bold).
   - Handle ("@brennan.sstools.co").
   - "286 followers · 359 following" stats row.
   - Tappable — navigates to the user's profile.
2. **Navigation rows** (each with a leading SF-Symbol-equivalent icon):
   - 🔍 Explore
   - 🏠 Home (highlighted as the active tab when on Home)
   - 💬 Chat
   - 🔔 Notifications
   - \# Feeds
   - ☰ Lists
   - 🔖 Saved
   - 👤 Profile
   - ⚙️ Settings
3. **Footer**
   - Terms of Service (link)
   - Privacy Policy (link)
   - Two pill buttons: **Feedback** (with chat icon) and **Help**.

The current SwiftUI drawer ships only a subset of the navigation rows and none of the header/footer chrome.

## Expected behavior

Full RN parity, in three zones:

- **Header**: avatar + display name + handle + followers/following counts. Tap navigates to Profile.
- **Nav rows**: nine rows in RN order — Explore, Home, Chat, Notifications, Feeds, Lists, Saved, Profile, Settings. The currently-selected tab gets the highlighted variant (bold + filled icon, matching RN). Each row has a leading icon and a label.
- **Footer**: Terms of Service link, Privacy Policy link, Feedback button (opens the feedback dialog or a referral sheet), Help button (opens the help-desk URL — same one wired into Settings → Help in #0123).

The drawer should slide over the active tab content with a partial-width overlay (~80% on phone, narrower on iPad/macOS) and dim the background. Tapping outside dismisses.

## Actual behavior

Drawer exists but has a reduced row set, no profile header, and no footer.

## Root cause

The drawer skeleton landed by #0072 was deliberately scoped down to the
"profile card on top, six nav rows, sign-out at the bottom" minimum so
the hamburger button had something to open. RN's
`Bluesky-ReactNative/src/view/shell/Drawer.tsx` is meaningfully richer
on every axis: the header has followers / following stats, the
navigation list is nine rows (not six) in a different order, and the
footer carries legal links plus Feedback / Help pill buttons. The
SwiftUI drawer never grew into that shape, so the iPhone navigation
experience was missing the most visible chrome of the official app.

## Fix

Promoted the previously private `helpDeskURL` constant in
`SettingsScreen.swift` into a public `BlueskyHelpURLs` namespace so the
Settings hub and the new drawer footer share one source of truth.
Added Terms of Service and Privacy Policy URLs to the same namespace —
both also live in RN's `Drawer.tsx` `ExtraLinks` block, so it makes
sense to keep all three together.

Added `Bluesky-SwiftUI/DrawerView.swift` (iOS-only) — a self-contained
overlay panel split into three zones that match RN top-to-bottom:

- **Header.** 64pt `AvatarView`, bold display name, secondary handle,
  and a stats row that reads "N followers · M following" with the
  numeric portion in semibold and the labels in the secondary text
  colour. `CompactNumberFormatter` (already used by the profile screen)
  formats the counts so `1234` reads as `1.2K`. The entire header is a
  button: tapping it switches the active tab to Profile and dismisses
  the drawer.
- **Nine nav rows in RN order.** Explore (→ Search tab), Home, Chat
  (→ Messages tab), Notifications, Feeds (→ My Feeds destination per
  #0073), Lists (→ Lists hub), Saved (→ Bookmarks), Profile, Settings.
  Each row has a leading SF Symbol and the active-tab variant renders
  with the filled icon and bold label, mirroring RN's
  `bold && a.font_bold` treatment. Notifications inherits the existing
  badge count and shows a brand-blue capsule when unread.
- **Footer.** Terms of Service and Privacy Policy as text links, then a
  solid Feedback pill and an outline Help pill. Feedback opens the
  GitHub Issues page in the browser as the pragmatic stop-gap until a
  real feedback flow exists; Help opens the shared
  `BlueskyHelpURLs.helpDesk` URL.

`MainTabView` keeps ownership of every navigation flag — `showMyFeeds`,
`showLists`, `showBookmarks`, `showSettings` — and threads them into
`DrawerView` as callbacks. Rows that need to push a non-tab destination
fire `onDismiss()` first, switch the tab, then sleep 220ms (matching
the slide-out animation duration) before flipping the destination flag,
exactly like the original inline drawer.

The existing `refreshViewerAvatar()` task already runs on the Profile
tab keyed by `session.currentAccount?.did`. It now also captures
`followersCount` / `followsCount` into two new state fields. The
drawer's `onAppearOnce` re-fires that task so the stats row is
populated even if the drawer opens before the initial keyed task has
run.

## Gotchas

- **Feedback pill is a GitHub Issues referral, not a feedback dialog.**
  RN's `Drawer.tsx` builds `FEEDBACK_FORM_URL` with the user's email
  and handle and opens the public Bluesky feedback Google Form. We
  don't have that form configured for this fork yet, and Privacy
  Policy explicitly forbids leaking email into URL parameters before
  consent. Pointing at `github.com/bluesky-social/social-app/issues`
  is a documented pragmatic stop-gap; replace once a real feedback
  flow lands. Tracked implicitly under this fix.
- **Stats row hides until hydrated.** When `viewerFollowersCount` is
  `nil` (drawer opens before `getProfile` returns) the row is skipped
  entirely rather than rendering "0 followers · 0 following". RN
  shows the row immediately because `useProfileQuery` hydrates from
  cache; without an equivalent profile cache plumbed into
  `MainTabView` (the `ProfileStore` lives inside `BlueskyProfile`)
  we'd flash zeros. The drawer's `onAppearOnce` callback re-fires the
  fetch on every open so the cold-start window is as short as the
  network round-trip.
- **Explore = Search tab.** RN's `SearchMenuItem` uses the label
  "Explore" but routes to the same `SearchTab` navigator that the
  bottom-bar magnifying glass owns. We map Explore → `selectedTab =
  .search` for the same reason. There is no separate Explore route in
  this build.
- **Lists row icon.** RN imports `BulletList_Stroke2_Corner0_Rounded`,
  which is a unique two-line bulleted-list glyph. The closest SF
  Symbol is `list.bullet.indent`; the more common `list.bullet` is
  visually too plain compared to RN. Swap if a custom asset gets
  imported.
- **Sign out moved off the drawer.** The original #0072 drawer carried
  a destructive "Sign out" row at the bottom; RN's Drawer.tsx does
  not. The sign-out affordance still lives in the Profile / Settings
  hub. Keeping the drawer to RN's exact zone count (header / nav /
  footer) avoids visual drift.
- **iPhone-only.** `DrawerView` is wrapped in `#if os(iOS)` because
  it uses `UIScreen.main.bounds.width` to size the panel. macOS uses
  the existing `NavigationSplitView` sidebar and doesn't need the
  drawer; iPad regular-width also uses the split view. If a unified
  drawer is needed later, swap the width source for a
  `GeometryReader`.
