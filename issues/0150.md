## Description

The hamburger-triggered sidebar drawer was added in #0072 with a small set of navigation rows. The React Native reference (attached) shows a much richer drawer with three distinct zones, and the SwiftUI version is missing most of them.

**RN drawer (top to bottom):**

1. **Profile header**
   - Avatar (circular, ~64pt).
   - Display name (large, bold).
   - Handle ("@brennan.sstools.co").
   - "286 followers · 359 following" stats row.
   - Tappable — navigates to the user's profile.
2. **Navigation rows** (each with a leading SF-Symbol-equivalent icon):
   - 🔍 Explore
   - 🏠 Home (highlighted as the active tab when on Home)
   - 💬 Chat
   - 🔔 Notifications
   - \# Feeds
   - ☰ Lists
   - 🔖 Saved
   - 👤 Profile
   - ⚙️ Settings
3. **Footer**
   - Terms of Service (link)
   - Privacy Policy (link)
   - Two pill buttons: **Feedback** (with chat icon) and **Help**.

The current SwiftUI drawer ships only a subset of the navigation rows and none of the header/footer chrome.

## Attachments

![Reference: RN sidebar drawer — full layout](rn-sidebar.png)

## Steps to reproduce

1. Run the SwiftUI app.
2. Tap the hamburger to open the drawer.
3. Compare to the attached RN screenshot.

## Expected behavior

Full RN parity, in three zones:

- **Header**: avatar + display name + handle + followers/following counts. Tap navigates to Profile.
- **Nav rows**: nine rows in RN order — Explore, Home, Chat, Notifications, Feeds, Lists, Saved, Profile, Settings. The currently-selected tab gets the highlighted variant (bold + filled icon, matching RN). Each row has a leading icon and a label.
- **Footer**: Terms of Service link, Privacy Policy link, Feedback button (opens the feedback dialog or a referral sheet), Help button (opens the help-desk URL — same one wired into Settings → Help in #0123).

The drawer should slide over the active tab content with a partial-width overlay (~80% on phone, narrower on iPad/macOS) and dim the background. Tapping outside dismisses.

## Actual behavior

Drawer exists but has a reduced row set, no profile header, and no footer.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/shell/Drawer.tsx` — the actual file with the layout source of truth. Contains the row order, icon set, header/footer, and styling tokens.
- File to edit: the drawer view added by #0072. Likely `Bluesky-SwiftUI/Bluesky-SwiftUI/MainTabView.swift` (for the inline overlay) or a sibling `DrawerView.swift`.
- "Explore" in RN's drawer is **not** the same as "Search" — it's a separate routing target. Confirm against `Drawer.tsx`. If they're synonyms in this build, mapping Explore → Search is acceptable, but note it in the resolution.
- "Feeds" in the drawer should route to the same My Feeds destination as the iOS top bar's `#` button (#0073).
- The Feedback button in RN opens a dialog with feedback options. **Pragmatic**: open a referral sheet pointing at GitHub Issues / a URL until a real feedback flow is built.
- The Help button should open the same `HELP_DESK_URL` constant used in #0123. Reuse the constant — don't duplicate it.
- This issue is the comprehensive drawer parity. If the work splits naturally during implementation (e.g., footer is trivial vs nav rows are substantial), file follow-ups; do not let a perfect-is-the-enemy-of-good prevent shipping the header + nav row chunks.
