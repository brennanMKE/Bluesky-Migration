# 0125 — List detail screen missing header (avatar, name, description, creator, action buttons)

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyLists |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

`ListDetailScreen` jumps straight to a tab strip (Members / Feed) with no header. The React Native reference (`Bluesky-ReactNative/src/screens/ProfileList/`) renders a full header above the tabs:

- List avatar (square thumbnail, ~64pt).
- List name (large title).
- Purpose badge (Curate / Moderation).
- Description text.
- Creator handle row ("by @alice.bsky.social", tappable to profile).
- Member count + subscriber count.
- **Primary action**: Subscribe / Unsubscribe (for mod lists) or Pin / Unpin (for curate lists).
- **Secondary actions menu** (ellipsis): Mute list, Block list (mod lists), Share list (copy link / sheet), Report list, Edit list (covered separately in #0127), Delete list (covered separately in #0127).

Without a header, users have no context about what list they're looking at and no way to subscribe/manage.

## Steps to reproduce

1. Tap into any list from `ListsScreen`.
2. Observe the screen opens directly to tabs with no header above them.

## Expected behavior

`ListDetailScreen` renders the full header above the tab strip. Action buttons are wired to the existing `ListsViewModel` mutations (mute, subscribe) and to a new edit/delete flow for #0127.

## Actual behavior

Just tabs.
