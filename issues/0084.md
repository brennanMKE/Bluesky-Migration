## Description

The profile info row (stats and URL) has four small differences from the React Native reference. Bundling because they all touch the same row layout and ship as one focused PR.

This is a follow-up to the iOS Profile parity audit in #0070.

## Scope

### 1. Stats order

- **Today**: `359 Following · 286 Followers · 1,424 Posts` — Following first.
- **RN**: `286 followers · 359 following · 1.4K posts` — Followers first.
- **Fix**: swap Following and Followers in the stats row.

### 2. Label casing

- **Today**: title-case ("Following", "Followers", "Posts").
- **RN**: lower-case ("following", "followers", "posts").
- **Fix**: lowercase the labels.

### 3. Count abbreviation

- **Today**: full-precision counts with group separators (`1,424`).
- **RN**: abbreviated counts (`1.4K`, `2.3M`, etc.).
- **Fix**: format counts using a compact-style number formatter (`NumberFormatter.Style.decimal` with `usesGroupingSeparator = false` plus a manual K/M/B suffix helper, or `Measurement`-based formatting). Threshold: show full precision below 1,000; `1.4K` between 1K–999K; `1.4M` above 1M.

### 4. URL display strips protocol

- **Today**: `https://brennan.sstools.co` rendered with the protocol prefix.
- **RN**: `brennan.sstools.co` — protocol stripped for display, but the link still opens with the full URL.
- **Fix**: strip the protocol prefix when rendering, retain it when handling the tap.

## Implementation notes

- Code lives in `BlueskyKit/Sources/BlueskyProfile/ProfileScreen.swift` (or its header view).
- The compact-number formatter belongs in `BlueskyUI` — extract `CompactNumberFormatter.string(from: Int)` and reuse it on follower/following counts wherever they appear in the app (also used in feed action bar repost/like counts under #0076).
- URL protocol stripping: a tiny helper `URL.displayString` that drops `https://` / `http://` and the trailing `/` if present.

## Acceptance

- Stats row reads `<followers> followers · <following> following · <posts> posts`.
- Counts are abbreviated as K/M for >= 1,000.
- URL renders without the protocol prefix; tapping still navigates to the full URL.
- iOS Simulator and macOS builds pass.
