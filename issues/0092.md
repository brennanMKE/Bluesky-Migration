# 0092 — Post-signup onboarding flow is missing

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyAuth / Bluesky-SwiftUI |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

After a successful create-account in the React Native client, the user is dropped into a multi-step onboarding flow at `Bluesky-ReactNative/src/screens/Onboarding/` that shapes the empty home feed into something useful. It includes (roughly): profile setup (display name + avatar pick), **interests** picker (multi-select of topic chips that seeds Discover), **suggested accounts** to follow (per-interest), **suggested feeds**, **suggested starter packs**, and a completion screen. The SwiftUI app has none of this — even if signup landed (#0091), a brand-new user would arrive at an empty Home with no recommendations.

This is also relevant for *existing* RN users who haven't been through onboarding yet, which RN handles via the `hasOnboarded` preference flag in `app.bsky.actor.preferences`.

## Steps to reproduce

1. (Once #0091 lands, or with a fresh test account) sign in as a new account that has not completed onboarding in RN.
2. Observe that the app goes straight to MainTabView with a blank/empty feed.

## Expected behavior

After successful signup (or after sign-in when `hasOnboarded` is false) the app presents an onboarding `NavigationStack` with:

1. Welcome / value-prop screen.
2. Profile setup: display name, avatar pick (PHPicker), short bio.
3. Interests picker: grid of topic chips (Tech, Sports, News, Art, etc.); multi-select; seeds Discover algorithm.
4. Suggested accounts: per-interest list of recommended profiles with Follow buttons.
5. Suggested feeds: per-interest list of feed generators with Follow / Pin buttons.
6. Suggested starter packs (optional — RN shows them).
7. Find friends (links to existing `FindContactsScreen`).
8. Completion: writes `hasOnboarded = true` to preferences, dismisses to `MainTabView`.

Each step has skip / back / next navigation. Pinned interests + selected follows are written via `app.bsky.actor.putPreferences` and `app.bsky.graph.follow`.

## Actual behavior

No onboarding UI at all. New accounts land on MainTabView with an empty feed and no recommendations.

## Notes

- RN reference: `Bluesky-ReactNative/src/screens/Onboarding/index.tsx` and `Steps/` siblings.
- Onboarding completion flag lives in preferences as `app.bsky.actor.defs#interestsPref` and a `hasOnboarded`-style flag — see RN's `state/preferences/onboarding.ts`.
- This is a Module 1 / Module 14 cross-cutting feature: needs `BlueskyAuth` for the gate, `BlueskySettings` for the preferences write, and (ideally) a new `BlueskyOnboarding` module to keep the screens isolated.
- Pairs with #0091 (signup) — together they form the full new-user journey.
