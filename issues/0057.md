# 0057 — Bookmarks not implemented: requires a server-side or local storage solution

| | |
|---|---|
| **Status** | resolved |
| **Module** | BlueskyFeed / BlueskyDataStore |
| **Platform** | All |
| **First seen** | 2026-04-30 |

## Description

The bookmark button exists in the `PostCard` action bar but is wired to a no-op (`a.onBookmark = { _ in }`). Bluesky's AT Protocol does not have a native bookmark lexicon — bookmarks must be implemented as either a private list (`app.bsky.graph.list` with a curatelist purpose) or local storage (e.g. SwiftData). Neither approach is currently implemented.

## Expected behavior

Tapping the bookmark button saves the post and it appears in a Bookmarks screen accessible from the Profile tab menu.

## Actual behavior

Tapping the bookmark button does nothing. There is no bookmarks screen with real data.

## Notes

Two implementation options:

1. **Server-side (private list)** — Create a private `app.bsky.graph.list` named "Bookmarks" on first use, then add/remove posts via `app.bsky.graph.listitem` records. Syncs across devices. More complex.

2. **Local storage (SwiftData)** — Store bookmarked post URIs (and optionally the full `PostView` snapshot) in a SwiftData model. Simple, no API required, but not cross-device.

Option 2 is simpler to ship first. `BookmarksScreen.swift` and `BookmarksViewModel.swift` exist but are backed by placeholder data.
