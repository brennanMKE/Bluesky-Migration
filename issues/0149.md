# 0149 — Raw rkey leaks into a feed/list row ("77a502dfc060" rendered as text)

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyFeed / BlueskyUI |
| **Platform** | macOS |
| **First seen** | 2026-05-07 |

## Description

In the macOS app, a feed/list row shows the feed name "Popular With Friends" alongside a raw 12-character hex string `77a502dfc060`. That string is the **rkey** of an `at://` URI (e.g. the trailing path component of `at://did:plc:.../app.bsky.feed.generator/77a502dfc060`) — internal storage detail that should never be user-visible.

This is a debug leak: somewhere in the feed/list row builder a developer-debug `Text(uri.rkey)` (or equivalent) is being rendered next to the proper display name.

The React Native iOS reference shows feed rows with name + creator handle + description + like count — never the rkey.

## Steps to reproduce

1. Run the macOS SwiftUI app.
2. Open the screen that lists feeds (likely **My Feeds**, **Saved Feeds**, or the Home feed-switcher row group).
3. Find the row for the "Popular With Friends" feed (or any custom feed).
4. Observe the rkey rendered as plain text next to the name.

## Expected behavior

The row shows the feed's `displayName` (and probably creator handle + description per the existing `FeedCard` shape from Module 4). No rkey, no CID, no DID, no AT URI.

## Actual behavior

Display name + raw 12-char rkey alongside it.

## Notes

- The leaking value `77a502dfc060` is the rkey of an `app.bsky.feed.generator` record. Search the SwiftUI tree for `uri.rkey`, `"\(uri" `, or any `Text` rendering of an `ATURI` to find the offender. Likely in `MyFeedsScreen.swift`, `SavedFeedsScreen.swift`, `FeedSwitcherView.swift`, or a `FeedCard` variant.
- A trailing relative timestamp ("5d") is visible in the same screenshot — that part appears to be intentional. The rkey next to the name is the bug.
- Audit the row builder for any other internal-only fields being leaked at the same time.
