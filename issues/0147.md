## Description

`PostEmbedView` renders a video embed as a still thumbnail with a play-icon overlay. Tapping it does nothing visible. The React Native client renders the video inline in feed (auto-playing muted when the cell is on screen, similar to Twitter / Instagram) and supports a fullscreen / lightbox playback mode on tap with player controls (scrubber, mute, time).

This is independent of the dedicated `VideoFeedView` (which has its own gaps in #0130) — it's the in-feed and in-thread playback path.

## Steps to reproduce

1. Open a post with a video embed in either feed or thread.
2. Tap the thumbnail.
3. Observe nothing happens (or no native player opens).

## Expected behavior

- Inline: render an `AVPlayerLayer`-backed video that auto-plays muted when the cell is visible (`onAppear` / `onDisappear` controls). Show a small mute icon overlay.
- Tap to open a **fullscreen player sheet** with native AVKit `VideoPlayer` controls (scrubber, time, mute, AirPlay, fullscreen-rotate).
- The sheet supports swipe-to-dismiss.
- Honor the user's "autoplay videos" preference (disable auto-play when off).

## Actual behavior

Static thumbnail with play-icon overlay; tap does nothing.

## Notes

- RN reference: `Bluesky-ReactNative/src/view/com/util/post-embeds/VideoEmbed.tsx`.
- File to edit: `BlueskyKit/Sources/BlueskyUI/PostEmbedView.swift`. Add a `VideoEmbedPlayer.swift` view wrapping `AVPlayer`/`VideoPlayer` for the fullscreen path.
- Preference: `autoplayVideos` — already in `SettingsViewModel`. Honor it.
- `VideoFeedView`'s gaps (mute toggle, action overlays) are filed at #0130; this issue is specifically about feed and thread embeds, not the dedicated video tab.

## Root cause

`PostEmbedView`'s video branch only rendered a static thumbnail with a play-icon overlay; tapping it had no handler attached, so users couldn't view a video at all from feeds or threads. The dedicated `VideoFeedView` already had the AVKit playback pattern (auto-play, looping, mute toggle), but that pattern was never lifted into the in-feed embed renderer.

## Fix

- Added `BlueskyUI/VideoEmbedPlayer.swift` — a self-contained inline `AVPlayer`-backed view that mirrors RN's `VideoEmbedInnerNative.tsx`:
  - Auto-plays muted on `onAppear` and tears down in `onDisappear` so each cell owns its `AVPlayer` lifecycle and the feed doesn't accumulate players as the user scrolls.
  - Loops via the `AVPlayerItemDidPlayToEndTime` notification (same trick as `VideoFeedView`).
  - Pins a small mute toggle to the top-trailing corner; default muted, like RN.
  - Honors a new `\.blueskyAutoplayVideos` SwiftUI environment value (defaulting to `true`). When the host injects `false` (from `SettingsViewModel.autoplayVideo`), the inline view falls back to the previous static-thumbnail-with-play-badge appearance and skips player creation entirely.
  - Tapping anywhere on the inline surface presents a fullscreen video sheet (`fullScreenCover` on iOS, `sheet` on macOS) with `VideoPlayer` *without* `.disabled(true)` so AVKit's native scrubber, time, mute, AirPlay, and picture-in-picture controls render. A "Done" button dismisses.
- Switched `PostEmbedView`'s `.video(let video)` branch to `VideoEmbedPlayer(video: video)` and removed the now-dead private `videoThumbnail(_:)` helper.

## Files changed

- `BlueskyKit/Sources/BlueskyUI/VideoEmbedPlayer.swift` (new)
- `BlueskyKit/Sources/BlueskyUI/PostEmbedView.swift` (use new player; drop dead thumbnail helper)

## Gotchas

- The inline `VideoPlayer` is wrapped in `.disabled(true)` so the system playback controls don't intercept the tap that opens the fullscreen sheet. The fullscreen sheet creates its own `AVPlayer` so dismissing tears down playback cleanly.
- Aspect ratio is capped at `max(ratio, 0.5)` (1:2) to match RN's `Math.max(aspectRatio, 1/2)` constraint in `VideoEmbed/index.tsx`, so portrait videos don't eat an entire feed cell.
- The host app should inject `.blueskyAutoplayVideos(settingsViewModel.autoplayVideo)` at the root once the wiring lands; until then the default of `true` matches RN's default behaviour.
