# 0110 — Message reactions (emoji) are not implemented

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskyMessages |
| **Platform** | All |
| **First seen** | 2026-05-06 |

## Description

The React Native chat client supports per-message emoji reactions: a user picks an emoji from a picker, the reaction attaches to the message, and a small reaction strip renders below the bubble showing each emoji and its count. Tapping the strip opens a dialog listing who reacted with what. RN exposes the picker via the per-message context menu (covered by #0106) and via a "+" affordance on long-press.

The SwiftUI client has no reactions UI: no picker, no rendered reaction strip, no dialog.

## Steps to reproduce

1. Open a thread where the other party (in RN) has reacted to one of your messages.
2. Observe the reaction is not shown in the SwiftUI bubble.
3. Try to add a reaction — there's no affordance.

## Expected behavior

- **Reaction strip**: under any bubble that has reactions, render a horizontal pill row with each unique emoji and its count (e.g. "👍 3 ❤️ 1").
- **Tap a pill**: opens a "Reacted by" sheet listing each user grouped under each emoji.
- **Long-press on bubble** (or context menu — see #0106): opens an emoji picker. Selecting one writes the reaction via `chat.bsky.convo.addReaction`.
- **Tap your own emoji in the strip**: removes the reaction (`chat.bsky.convo.removeReaction`).

## Actual behavior

No reactions UI at all; reactions present on incoming messages are dropped silently.
