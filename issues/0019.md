# 0019 — Module 7 feature: reply composer not implemented

| | |
|---|---|
| **Status** | resolved |
| **Module** | BlueskyFeed |
| **Platform** | All |
| **First seen** | 2026-04-29 |

## Description

Tapping the reply button on a post in `ThreadView` (or resolving issue 0004 for the feed) should open a composer pre-filled with the parent post context. No such composer integration exists in `ThreadView`.

## Steps to reproduce

1. Navigate to any post thread.
2. Tap the reply button on any post in the thread.

## Expected behavior

`ComposerSheet` opens with the parent post's URI and CID pre-filled as a reply reference. Submitting posts the reply and it appears in the thread.

## Actual behavior

No reply composer is wired into `ThreadView`. The reply button is either absent or a no-op.

## Notes

`ComposerViewModel` and `ComposerSheet` already exist in `BlueskyComposer`. `ThreadView` needs to present `ComposerSheet` with a `replyTo: PostRef?` parameter. `PostRecord` should carry the `reply` field (`ReplyRef` with `parent` and `root`).
