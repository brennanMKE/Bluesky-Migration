# 0090 — iOS Search screen needs slim top bar to match Home / Notifications

| | |
|---|---|
| **Status** | open |
| **Module** | BlueskySearch / Bluesky-SwiftUI |
| **Platform** | iOS |
| **First seen** | 2026-05-06 |

## Description

The Search tab still uses a giant left-aligned "Search" headline taking ~25% of the screen — the same anti-pattern that #0072 (Home) and #0077 (Notifications) already fixed for their respective tabs. To match the React Native reference and feel consistent with Home and Notifications, Search needs the same slim top-bar treatment: hamburger / inline title / appropriate trailing action.

## Attachments

![Search screen — giant headline; needs slim top bar](search.png)

## Steps to reproduce

1. Run the SwiftUI iOS app.
2. Tap the Search tab.
3. Observe the giant "Search" headline above the search field.

## Expected behavior

A slim top bar flush below the status bar:
- **Left**: hamburger button — opens the same shared drawer used by Home (#0072) and Notifications (#0077).
- **Center**: "Search" title in a single line (regular weight; matches the wordmark sizing on Home / Notifications).
- **Right**: leave empty, or use whatever trailing action RN puts there (likely none — the search field handles all the discovery affordances).

The search field stays where it is, sitting directly below the slim top bar.

## Actual behavior

A giant `.large` "Search" navigation title takes ~25% of the vertical space above the search field.

## Implementation notes

- Reuse `BlueskyTopBar` (added in #0072) — three slots, drop the trailing slot or leave it blank.
- Wire the hamburger to the existing `showDrawer` state in `MainTabView` (the drawer is already shared across Home and Notifications).
- Search is already iOS+macOS — this change is iOS only (macOS keeps its existing toolbar). Gate with `#if os(iOS)`.

## Acceptance

- iOS Search screen has a slim top bar (hamburger / "Search" / empty) flush below the status bar.
- Hamburger opens the shared drawer.
- macOS Search unchanged.
- Combined with #0088 (top safe-area gap), the bar should fill behind the status bar.
